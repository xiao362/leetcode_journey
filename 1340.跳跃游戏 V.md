# 方法一：dfs+记忆化

普通 BFS 适合无权图的**最短路径**，因为谁先扩展到谁就最短。但这道题求的是**最长路径**，普通 BFS 先访问到的不一定是最优的，一个节点可能被多次更新，最终变成暴力搜索。

## 解题思路和过程

通过题目给的条件，以及对示例数据的模拟，在这个过程中，我们知道这是一个让我们找最长路径的问题，那我们最起码要把每一个节点都走过一遍，因为是找最长的路径，我们可以用dfs来找，因为**dfs的特点是一条路走到黑，适合找出最长的一条路**

在常见的dfs中我们用的一般是将vis设置为bool型，这样同样可以走过所有情况，但是因为各种路都到走到底，就导致会走一些重复的路，且在这种情况下基本让时间复杂度来到了$O(n^2)$，如果在加上分别以各个点为起点的一次遍历，时间复杂度会来到$O(n^3)$，n最大为1000，	这也意味着倘若是$O(n^2)$我们还能接受但$O(n^3)$就肯定会爆时间了

故我们要**改变记忆化的形式，在记录是否来过的同时，记录当前位置能够跳跃的最大次数（有点像动态规划）**，在确定方法后就下来就是代码实现的问题了

首先是每个数能跳跃的边界问题，我们要**保证左边界或者右边界，要在数组边界里面**，这倒是好解决可以用**`left=max(0,idx-d);`,`right=min(n-1,idx+d);`**,来确定边界，再观察跳跃条件：

从下标 `i` 跳到下标 `j` 需要满足：`arr[i] > arr[j]` 且 `arr[i] > arr[k]` ，其中下标 `k` 是所有 `i` 到 `j` 之间的数字（更正式的，`min(i, j) < k < max(i, j)`）。

这也**意味着在`i`与`k`之间倘若遇到大于`arr[i]`的值`arr[j]`就不能成立**，那我们可以**从`i`出发分别向两边走遇到不能跳的后面的就直接break，其他的肯定不能跳跃了**

因为使用了记忆化所以最大跳跃次数不超过1000次**可以直接用递归**，另外用**cnt记录在dfs中可能会遇到的最大跳跃次数**

最后用遍历每一个起点，用**ans记录最大跳跃次数**

## 复杂度

- 时间复杂度:$O(n*d)$
- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    int maxJumps(vector<int>& arr, int d) {
        int n = arr.size();
        int ans = 0;
        vector<int> vis(n, 0);

        auto dfs = [&](auto&& self, int idx) {
            if (vis[idx])
                return vis[idx];

            int left = max(0, idx - d);
            int right = min(n - 1, idx + d);

            int cnt = 0;
            for (int i = idx - 1; i >= left; --i) {
                if (arr[idx] > arr[i]) {
                    cnt = max(cnt, self(self, i));
                } else
                    break;
            }
            for (int i = idx + 1; i <= right; ++i) {
                if (arr[idx] > arr[i]) {
                    cnt = max(cnt, self(self, i));
                } else
                    break;
            }
            return vis[idx] = cnt + 1;
        };

        for (int i = 0; i < n; i++) {
            ans = max(ans, dfs(dfs, i));
        }
        return ans;
    }
};
```
