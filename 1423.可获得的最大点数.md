# 方法一：滑动窗口（求最大）

两个方法各有优缺而无优劣，这个方法或许更适合处理k<n/2的情况，而第二个方法则反之，另外如果把两边的元素看成两个方向，用bfs或者dfs来求解也是容易想到的方法，但重复数据太多毫无疑问会超时

## 解题思路和过程

题目要求我们，每次从两边拿牌，这也就意味着我们的牌是**“相对连续“的（并不是大小上的连续，而是座标上的连续，且头和尾也算是连续的一种情况，就像是一个环一样**），且还要求我们牌的数量一定为k，像这种数量是一定的，且元素是连续的，并且然我们统计区间内的某某个数或者全部数之和等等，我们很容易想到定长滑动窗口，但这个比较特殊他并不是绝对连续的，为了让数据”连续”的被统计，我们可以观察数据的统计方式，我们发现**有三种可能面对最大值的情况：**

> 1. **全取得左端，右边一个没取**
> 2. **全取得右端，左边一个没取**
> 3. **两端各取一点**

这时我们或许可以观察到一些关系，比如**第一种情况如果少取一个，那么最右端就要取走一个，这是就变成了第三种情况，如果一直这样就会变成第二种情况，又因为最大值只会在这三种情况中选出**，我们直接遍历完这三种情况，取最大值即可（变相的滑动窗口），关于代码实现如下：

我们**设置sum存储“窗口”的数据的和，设置`left=k-1`,方便不断将左端的值抛出“窗口”,设置`right=n-1`不断加入新值，一开始可以遍历0到k，直接将第一种情况作为初始数据统计出来，然后用`right>=n-k`来控制边界，毕竟不能走过了，期间不要忘了用ans更新最大值即可**



## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int maxScore(vector<int>& cardPoints, int k) {
        int n=cardPoints.size();
        int ans=0;
        int sum=0;
        int left=k-1;
        int right=n-1;
        for(int i=0;i<k;++i){
            sum+=cardPoints[i];
        }
        ans=sum;
        while(right>=n-k){
            sum-=cardPoints[left--];
            sum+=cardPoints[right--];
            ans=max(ans,sum);
        }
        return ans;
    }
};
```



# 方法二：滑动窗口（求最小）

## 解题思路和过程

通过前面过程的分析，以及上一个解法的模拟，我们是否可以**改变一种思路，我们是否可以通过求某一段连续区间的最小值，来反过来求最大值呢**，答案是肯定的，因为区间是定长的，又因为**最大值所用的数据只能在两端取**，这次我们**不观察可能取最大值的数据，我们看除他们之外的值**，如果用上一个方法模拟。我们发现，**他作为一个连续定长（长度为n-k）的区间，从最右端走到了最左端，这恰是一个标准的定长滑动窗口问题**，我们**求得他们得最小值，然后用整个数组得和减去最小值，就可以得到最大值**了，且最大值也**符合上一个方法得要求**，具体是实现如下：

用res统计整个数组的和，用sum统计“窗口“数据的和，用minval不断更新最小值，然后从头遍历，长度不够n-k则不参与统计，就这样最后返回res-minval即可

> 注意：n==k，minval就没有意义，直接跳过就行，并且在返回res-minval时别忘了minval最开始初始化为了INT_MAX,可以把它变为0，或者改变返回值即可

## 复杂度

- 时间复杂度:$O(n)$

- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int maxScore(vector<int>& cardPoints, int k) {
        int n=cardPoints.size();
        long long res=0;
        int minval=INT_MAX;
        int sum=0;
        int cnt=n-k;
        for(int i=0;i<n;++i){
            sum+=cardPoints[i];
            res+=cardPoints[i];
            if(i<cnt-1||cnt==0)continue;
            minval=min(minval,sum);
            sum-=cardPoints[i-cnt+1];
        }
        if(n==k)minval=0;
        return res-minval;
    }
};
```
