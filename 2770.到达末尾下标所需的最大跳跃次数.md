# 方法一：dfs+记忆化

观察题目给的跳跃的条件：

- *`-target <= nums[j] - nums[i] <= target`*
- *`0 <= i < j < n`*

即只要是在他后边的且当前数的差值的绝对值小于等于target即可发生跳跃，通过模拟题目给的示例我们可以发现，跳跃是有方向的（由前致后），且对于同一个nums[i]，他是**可以有多个nums[j]与之对应可以发生跳跃的**，为了找到最大跳跃次数，我们可以想到的最直观的方法就是暴力dfs搜索，只要是能到达终点的就是一个合理的答案，否则就是-1

附上暴力bfs代码：

```c++
class Solution {
public:
    int maximumJumps(vector<int>& nums, int target) {
        int n=nums.size();
        vector<int> arr[n];
        for(int i=0;i<n;++i){
            for(int j=i+1;j<n;++j){
                if(abs(nums[j]-nums[i])<=target){
                    arr[i].push_back(j);
                }
            }
        }
        int ans=-1;
        int res=0;
        
        auto dfs= [&](auto self,int i)-> int{
            if(i==n-1){
                ans=max(ans,res);
            } 
            for(auto p:arr[i]){
                res++;
                self(self,p);
                res--;
            }
            
            return 0;
        };
        dfs(dfs,0);
        return ans;
    }
};
```

毫无疑问：在没有记忆化搜索的情况下，这无异是会爆时间的

## 解题思路和过程

为了舍弃一些反复的过程，我们需要添加记忆搜索，且为了便于得出最终的结论（最大跳跃次数问题）我们可以开一个int类型的memo数组来记录已经走过的路到最后终点的最大跳跃次数，这点思想**类似于动态规划，即memo[i]存的是从i到终点的最大次数**，这也意味着***memo[n-1]要初始化为0***，因为**dfs在整个过程虽是正向进行的，但memo的更新确是倒着进行的，故次数是从后往前加的**

同时题目要求到达不了终点的值为-1，又为了区分某个位置是否到达过，所以将**memo初始化为-2（多少都行），表示还没有走过这里**，因为我用的是lambda写的dfs，所以**提醒一句要注意lambda写递归是要注意调用自身**

## 复杂度

- 时间复杂度:O($n^2$),最坏
- 空间复杂度:O($ n^2 $)，可以优化为O(n),就是在搜索时把后面符合条件的枚举出来，不用存了，但这样或许好想且容易理解

## Code

```C++
class Solution {
public:
    int maximumJumps(vector<int>& nums, int target) {
        int n=nums.size();
        vector<int> arr[n];
        for(int i=0;i<n;++i){
            for(int j=i+1;j<n;++j){
                if(abs(nums[j]-nums[i])<=target){
                    arr[i].push_back(j);
                }
            }
        }
        
        vector<int> memo(n,-2);
        memo[n-1]=0;
        auto dfs= [&](auto self,int i)-> int{
            if(memo[i]!=-2)return memo[i];
            else{
                int maxval=-1;
                for(auto p:arr[i]){
                    int res=self(self,p);
                    if(res!=-1){
                        maxval=max(maxval,res+1);
                    }
                }
                return memo[i]=maxval;
            }
            
            
        };
        
        return dfs(dfs,0);
    }
};
```



# 方法二：dp动态规划

这个方法，在实操上手之前，我总是犹犹豫豫想着，想着怎样尽可能地，减少回溯的次数，一直在考虑怎样才能达到最优解，但是不考虑这点直接上手才发现，这代码可比上面的好实现多了

## 解题思路和过程

按照传统的动规思想结合题目，最容易想到的***dp状态就是记dp[i]是起点到i点的最大跳跃次数***，这里为了保证dp的值是最大跳跃次数，我们需要开两个循环，因为dp[i]状态变化并不依赖dp[i-1],而是依赖dp[j] (满足跳跃条件)，这里如果能像上面记忆化一样，倒是可以少遍历一些数，但这里我们就直接将i前面的数跑一遍，找到最大值，遍历完就直接出结果了

但是不要忘记不能到达的终点要返回-1，你可将***dp初始化 $$-\infty$$(最好初始化为$$ -\infty$$，不然结果最小就不是-1了)***，都行只要能区分开是否能到达终点就行



## 复杂度

- 时间复杂度:O($$n^2$$)
- 空间复杂度:O(n)

## Code

```
class Solution {
public:
    int maximumJumps(vector<int>& nums, int target) {
        int n=nums.size();
        vector<int> dp(n,INT_MIN);
        dp[0]=0;
        for(int i=0;i<n;++i){
            for(int j=0;j<i;++j){
                if(abs(nums[i]-nums[j])<=target){
                    dp[i]=max(dp[i],dp[j]+1);
                }
            }
        }
        return dp[n-1] < 0 ? -1 : dp[n-1];
    }
};
```
