# 方法一：双指针

## 解题思路

因为这道题我们同时面对四个数，倘若我们用暴力做法，直接开四个指针，把每一种情况都比枚举出来时间复杂度来到$O(n^4)$，毫无疑问这太大了，观察数据

- `1 <= nums.length <= 200`
- `-109 <= nums[i] <= 109`
- `-109 <= target <= 109`

我们发现倘若时间复杂度是$O(n^3)$，那么题目是可以接受了，也就是说我们最少优化掉$O(n)$的复杂度

观察数组是无序的，在没有规律可言的情况情况下（题目越短越是麻烦）我们除了枚举也不会有其他办法了，为避免这种情况我们的第一步肯定是排序

**为了避免同时操作和四个指针的复杂情况我们采取固定两个数**，而**同时操作另外两个数的办法**，在原数组排序后的基础上，用两个循环把每个数依次作为固定值，这时问题就转化为了：**两数之和最接近`target`的问题**，这个问题我们可以**用双指针来解决**，即用数组两端的双指针分别作为我们要找的另外两个值，因为双指针的时间复杂度是$O(n)$，恰好满足我们的要求，这时会有下面的情况：

如果二者和偏大，**这时我们就要让右指针左移，直到我们发现二者的和小于`target`了，这时我们发现，right右边的值是肯定不会成为我们需要的值，因为他们加最小值都大于`target`，然后接着如果我们想要等于`target`，我们就要使二者的和变大，也就是左指针右移（右指针右边的已经淘汰），直到二者和再次大于`target`，此时类似的left左边的也已经被淘汰，因为他们加上数组可能值的最大值后还是小于`target`**，以此类推，我们**一定会遇到让二者和最接近`target`的情况**

然后通过不断改变固定值就可以所有情况下和会等于0的答案了

## 实现细节

1. 两个指针采取两层循环固定的形式，为了避免重复计算两个固定值为一前一后的形式
2. 遇到四者指针坐标相等的情况可以跳过该次
3. 在等于target填入答案时，同时缩小左右指针，因为只缩小一边，只有相同的值才能再次令和为`target`，我们不要相同的值，故缩小时我们还要搭配while一直缩小到不相同的值或者两指针相遇为止
4. 固定值以及做右指针没有顺序要求，也就代表我们没必要每次都让left从0出发，可以让他从固定值的右边出发，（同时这也是**为了保证不出现相同的解**）
5. 相同的固定值我们只需要计算一遍，没必要重复计算
6. 为保证四者坐标不相同，固定值到n-3的位置就可以结束了，后面是另一个固定值还有left以及right的位置

## 复杂度

- 时间复杂度:$O(n^2)$，严谨的来说是$O(n^2)+O(n \log n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        sort(nums.begin(),nums.end());
        int n=nums.size();
        vector<vector<int>> ans;
        for(int i=0;i<n-3;++i){
            if(i>0&&nums[i]==nums[i-1])continue;
            for(int j=i+1;j<n-2;++j){
                if(j>i+1&&nums[j]==nums[j-1])continue;
                int left=j+1;
                int right=n-1;
                while(left<right){
                    long long sum=(long long)nums[i]+nums[j]+nums[left]+nums[right];
                    if(sum==target){
                        ans.push_back({nums[i],nums[j],nums[left],nums[right]});
                        left++;
                        right--;
                        while(nums[left]==nums[left-1]&&left<right)left++;
                        while(nums[right]==nums[right+1]&&left<right)right--;
                    }else if(sum>target){
                        right--;
                    }else{
                        left++;
                    }
                }
            }
        }
        return ans;
    }
};
```



## 优化

### 优化细节

1. 因为数组是有序的，所以他倘若固定值以及他后边的两位的他们的和已经大于target后面就没必要比较了，差值只会越来越大
2. 类似的，固定值加上数组最大的两位还是小于target，那么小于这两位的也就没必要计算了

```C++
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        sort(nums.begin(),nums.end());
        int n=nums.size();
        vector<vector<int>> ans;
        for(int i=0;i<n-3;++i){
            if(i>0&&nums[i]==nums[i-1])continue;
            if((long long)nums[i]+nums[i+1]+nums[i+2]+nums[i+3]>target)break;
            if((long long)nums[i]+nums[n-3]+nums[n-1]+nums[n-2]<target)continue;
            for(int j=i+1;j<n-2;++j){
                if(j>i+1&&nums[j]==nums[j-1])continue;
                int left=j+1;
                int right=n-1;
                if((long long)nums[i]+nums[j]+nums[j+1]+nums[j+2]>target)break;
                if((long long)nums[i]+nums[j]+nums[n-1]+nums[n-2]<target)continue;
                while(left<right){
                    long long sum=(long long)nums[i]+nums[j]+nums[left]+nums[right];
                    if(sum==target){
                        ans.push_back({nums[i],nums[j],nums[left],nums[right]});
                        left++;
                        right--;
                        while(nums[left]==nums[left-1]&&left<right)left++;
                        while(nums[right]==nums[right+1]&&left<right)right--;
                    }else if(sum>target){
                        right--;
                    }else{
                        left++;
                    }
                }
            }
        }
        return ans;
    }
};
```
