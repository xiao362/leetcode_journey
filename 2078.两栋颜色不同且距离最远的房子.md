#方法一：贪心

#解题思路和过程

在房子颜色为colors = [1,1,1,6,1,2,1]时，在最容易想的暴力解法就是，**用两个指针，用两个正向循环将每种情况都考虑**，但是毫无疑问这个算法的**时间复杂度是O(n^2)**，为了优化这个算法我们是不是可以观察这个过程中可以省略的地方呢

我们发现当一个索引为0，另一个索引为5时，**我们可以发现中间步骤完全不需要判断**

> 为了实现这一点我可以将，循环倒过来跑，当第一次遇到不同的颜色时就可以结束了

但是我们发现时间复杂度还是没有降下来，毫无疑问，主要是我们还有一个指针在正向跑，不断压缩区间的大小，如果我们想要去掉这个指针，那么我们要考虑什么呢，那就是**如果两个不同的房子确定了最大距离，那么是否有一个房子一定在边界，我想这是肯定的**这可以分两种情况讨论：
为什么最大距离一定涉及至少一个边界房子？

> - 假设存在一对房子 (i, j) 使得 colors[i] != colors[j] 且距离 |i - j| 最大。如果这对房子中既没有左边界 0 也没有右边界 n-1，即 0 < i ≤ j < n-1，那么我们可以尝试向两端扩展：
- 若 colors[0] != colors[j]，则 (0, j) 的距离大于 (i, j)，与假设的最大性矛盾。
- 若 colors[0] == colors[j]，则必然有 colors[i] != colors[0]（因为 colors[i] != colors[j]）。此时 (i, n-1) 的距离大于 (i, j)，同样推出矛盾。
- 因此，最大距离的配对中必然包含 左边界 0 或 右边界 n-1。基于这一性质，我们只需分别固定首、尾元素，各做一次线性扫描寻找最远的异色房子，取两次结果的最大值即可。时间复杂度降为 O(n)。


##复杂度

- 时间复杂度：O(n);
- 空间复杂度：O(1);

##Code

```
class Solution {
public:
    int maxDistance(vector<int>& colors) {
        int tail=colors.size()-1;
        int ans=0;
        for(int i=tail;i>=0;--i){
            if(colors[0]!=colors[i]){
                if(i>=tail/2){
                    ans=max(i,ans);
                }else{
                    ans=max(ans,tail-i);
                } 
            }  
        }
        return ans;
    }
};```
