# 方法一：双指针

## 解题思路

题目要求我们求出数组中所有满足`0 <= i < j < n`，且`lower <= nums[i] + nums[j] <= upper`的个数，如果我们用暴力的办法来做这道题，时间复杂度是$O(n^2)$会超时，但是在数组每个数没有规律的情况下，暴力才是唯一解，倘如我们想要优化，排序就是必要步骤，排序后我们如果用双指针来做这道题，会发现我们并不能应对完全统计好每一种情况，往往会顾此失彼，保证了lower就无法同时保证upper，想同时满足两个条件我们就要回退指针，这也就失去双指针的意义了

所以我们用逆向思路，既然我们的双指针，一般保证的是单边的大于或小于那我们是否通过求其他的来间接求取满足在两边之间的情况呢，下面有两种思路：

1. 因为每一对情况都考虑的个数是`n*(n-1)/2`，我们可以通过求小于`lower`以及大于`upper`通过相减来求出在两边之间的
2. 我们可以求出大于等于`lower`的情况个数以及大于`upper`的情况个数，同样通过相减求出在两者之间的（反过来求小于`lower`和小于等于`upper`相减也一样）

## 复杂度

- 时间复杂度:$O(n\ log n)$，主要是排序消耗
- 空间复杂度:$O(1)$

## 代码细节

1. 要注意：起初求ans时int可能会溢出，所以用long long

### 思路一

> 二者满足条件后增加的个数都是`cnt+=right-left`

## Code

```C++
class Solution {
public:
    long long countFairPairs(vector<int>& nums, int lower, int upper) {
        long long n=nums.size();
        long long ans=n*(n-1)/2;
        sort(nums.begin(),nums.end());
        int left=0;
        int right=n-1;
        long long cnt=0;
        while(left<right){
            if(nums[left]+nums[right]>upper){
                cnt+=right-left;
                right--;
            }else{
                left++;
            }
        }
        ans-=cnt;
        left=0;
        right=n-1;
        cnt=0;
        while(left<right){
            if(nums[left]+nums[right]<lower){
                cnt+=right-left;
                left++;
            }else{
                right--;
            }
        }
        ans-=cnt;
        return ans;
    }
};
```

### 思路二

> 如果直接按照上边思路直接写，难免还是写两个函数，有点麻烦，我们可以直接写一个大于（小于）等于的函数，在函数值时通过加一或者减一，来去掉一边等于的情况，减少代码量

写法一（upper）：

```C++
class Solution {
public:
    long long countFairPairs(vector<int>& nums, int lower, int upper) {
        ranges::sort(nums);
        int n=nums.size();
        auto count=[&](int upper){
            long long cnt=0;
            long long left=0;
            long long right=n-1;
            while(left<right){
                if(nums[left]+nums[right]<=upper){
                    cnt+=right-left;
                    left++;
                }else{
                    right--;
                }
            }
            return cnt;
        };
        return count(upper)-count(lower-1);
    }
};
```

写法二（lower）：

# 方法二：三指针

本质上于双指针思路相同

## 解题思路

先定义一个指针是两数相加时较小数的坐标，定义两个指针`l`表示于左边指针相加后恰好刚刚大于等于lower的坐标，`r`表示于左指针相加后恰好刚刚大于upper的坐标，这两个指针中间所有元素就是满足在lower于upper中间的数

本质是将上面的函数操作融合到一个循环中

## 实现细节

1. 注意对取结果时`l`与`r`分别于`i`取max，因为`i`是个会走到头的指针，while里的条件不能一直保证满足条件

## 复杂度

- 时间复杂度:$O(n \log n)$排序消耗
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    long long countFairPairs(vector<int>& nums, int lower, int upper) {
        ranges::sort(nums);
        int n=nums.size();
        int l=n-1;
        int r=n-1;
        long long ans=0;
        for(int i=0;i<n;++i){
            while((long long)nums[i]+nums[l]>=lower&&l>i){
                l--;
            }
            while((long long)nums[i]+nums[r]>upper&&l<r){
                r--;
            }
            ans+=max(i,r)-max(i,l);
        }
        return ans;
    }
};
```

# 方法三：二分

## 解题思路

在前面两个思路上进一步优化，我们不用指针查，而是用二分（性能略差），同样我们开一个指针作为两数中较小的那一个`num[i]`

## 实现细节

1. 要求大于`upper-num[i]`的第一个数，不是大于等于，所以用`upper_bound`求右边界
2. 因为坐标不等相同，所以二分的起点应在`i`之后，也就是`nums.begin()+i+1`
3. `upper_bound()`与`lower_bound()`返回的都是地址

## 复杂度

- 时间复杂度:$O(n\log n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    long long countFairPairs(vector<int>& nums, int lower, int upper) {
        ranges::sort(nums);
        long long ans=0;
        for(int i=0;i<nums.size();++i){
            auto l=lower_bound(nums.begin()+i+1,nums.end(),lower-nums[i]);
            auto r=upper_bound(nums.begin()+i+1,nums.end(),upper-nums[i]);
            ans+=r-l;
        }
        return ans;
    }
};
```
