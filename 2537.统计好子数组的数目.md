# 方法一：滑动窗口

## 解题思路

题目要求我们统计如果有 **至少** `k` 对下标 `(i, j)` 满足 `i < j` 且 `arr[i] == arr[j]`的子数组的个数，因为我们的研究对象是子数组，即可以看作是一个窗口，且窗口要满足的条件可以量化，那我们就可以用滑动窗口的思路来解决

首先就是根据数学规律，当区间内有n个相同元素时有$$n(n-1)/2$$对坐标满足条件，但是我们需要把满足条件的对数对应到每次增加的满足条件的对数也就是n-1个，因为我们是计算机，上面相当于累加求和公式，但我们不需要公式我们需要的就是每次增加的个数，方便我们明确当前窗口的状态

我们让**窗口处于恰好不满足条件的状态**，即`[left,right]`是不满足条件的，但`[left-1,right]`，`[left-1,right]`，`[left-3,right]`......`[0,right]`都是满足条件的，**个数也就是left个**

## 实现细节

1. 注意返回值是long long类型
2. 我们统计窗口内某个元素的个数，不在乎顺序啥的，直接用`unordered_map<int,int> mp`统计个数
3. 每多一个元素增加`cnt+=mp[nums[i]]-1`，减少也类似
4. 用while不断维护区间，**使一直处于恰好不满足条件的状态**

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    long long countGood(vector<int>& nums, int k) {
        int n=nums.size();
        long long ans=0;
        int left=0;
        long long cnt=0;
        unordered_map<int,int> mp;
        for(int i=0;i<n;++i){
            mp[nums[i]]++;
            cnt+=mp[nums[i]]-1;
            while(cnt>=k){
                mp[nums[left]]--;
                cnt-=mp[nums[left++]];
            }
            ans+=left;
        }
        return ans;
    }
};
```
