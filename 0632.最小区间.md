# 方法一：滑动窗口

## 解题思路和过程

首先题目给我们了**k个区间**，题目要求我们**从这几个几件内的数找到一个范围（窗口），使得窗口内至少这k个区间里每个区间至少要有一个数，并且要求我们找这个区间的最小范围**

首先题目给我们的数据是一个二维数组，我们要找窗口来包含这几个区间里的元素时，肯定涉及到大小的比较，还句话说倘若数组元素是有序的肯定比较好处理些，毕竟我们找窗口的最小值，便相当于找窗口左边与有边界的最小差值，虽然数组本身有序但是，由于数组是二维数组，我们在取窗口的最小值时不方便，甚至需要反复比较几个区间的最小值，那么这时我们就可以**将二维变为一维数组**，只是**这时虽然边界好比较了，但我们失去了某个元素元素本是来源于那个区间的信息**，这时最小区间依旧不易确定，故为了在排序时保留他来自那个区间的信息，我们可以**开一个结构体来存储大小信息以及区间信息**

存好数据并且排完序后我们就需要，来找区间的大小了，这种一位数组上根据条件找范围，我们可以使**用滑动窗口来解决（可以遍历到所有满足条件的区间）**，而滑动窗口**关键就是边界缩小的条件**，我们要保证每个区间都至少有一个数，我们可以**用check数组，当某个区间的数进来一次我们就让check对应区间++**，这时只要满足所有check值都不为零就证明当前窗口满足条件了，但如果我们每次都要把check所有数都遍历一遍才能判断窗口是否满足条件，这无疑是一笔不小的开销，这里我们可以注意到，这与数有关的多个并集的问题我们可以**用checked来存已经存在数的区间个数，只要checked==n，就代表窗口满足条件**

在满足条件后我们**要更新答案ans还要使区间长度最小，所以我们用cnt来记录最短区间长度**

## 复杂度

- 4  时间复杂度:$O(k*logk)$**，k表示数组的元素个数，这主要是排序的消耗**，滑动窗口的时间复杂度是$O(k)$
- 空间复杂度:$O(k)$

## Code

```C++
class Solution {
public:
    struct idx{
        int x,y;
    };

    vector<int> smallestRange(vector<vector<int>>& nums) {
        int n=nums.size();
        vector<idx> p;
        vector<int> len(n,0);
        vector<int> ans(2,0);
        vector<int> check(n,0);
        int checked=0;
        int left=0;
        int right=p.size();
        int cnt=INT_MAX;

        for(int i=0;i<n;++i){
            len[i]=nums[i].size();
            for(int j=0;j<len[i];++j){
                p.push_back({nums[i][j],i});
            }
        }
            
        sort(p.begin(),p.end(),[](const idx& a,const idx& b)->bool{
            return a.x<b.x;
        });
        
        for(int i=0;i<right;++i){
            check[p[i].y]++;
            if(check[p[i].y]==1)checked++;
            while(checked==n){
                check[p[left].y]--;
                if(check[p[left].y]==0)checked--;
                if(p[i].x-p[left].x<cnt){
                    cnt=p[i].x-p[left].x;
                    ans[0]=p[left].x;
                    ans[1]=p[i].x;
                }
                left++;
            }
        }
        return ans;
    }
};
```

# 方法二：堆

## 解题思路和过程

在上面思路的基础上，**这次我们不再把两个数组合并到一起**，上面因为要后面排序，所以我并没有强调，**原数组的每个区间都是非递减序列的性质，**这次我们利用这点性质，就像前面说不将数组合并不好判断大小的说法，我们可以**通过`tuple<int,int,int>`** 来解决这个问题，**第一个值存对应得nums值，后面分别存`i`和`j`，这样当我们取出最小值的时候，就能找到他对应的区间以及位置了**，同时也为不每次都遍历一遍找最小值，我们**用小根堆直接存数据，取出来就是最小值**

上面基本上也就是实现的大致过程，我来进一步解释一下这样为什么能找到最小区间，首先**我们将每个数组的值都存进去了一个，在取出来后，又通过tuple带的位置再把同一区间的下一个值存进去，这就保证了小根堆里一直满足条件**，这样我们就**只需要统计最小值以及最大值，并更新使区间最小的边界值即可**，另外我们每次都是使小根堆里的最小值出来，使得每次都是对应数据的最优解（**这里有贪心的思想，实在不懂，就回忆滑动窗口的过程，这两者类似**）

过程中用r，l所存数据的最大以及最小值，ans_r,ans_l，对应最优解

> 关于结束条件：j+1==nums[i].size()，就是取出数的对应区间的数已经取完了，没办法加入新值了，那后面就要少这个区间的值了，也就没比要进行下去了

## 复杂度

- 时间复杂度:$O(k*logn)$，k是nums元素个数，n是nums长度，logn是堆操作的时间

- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    vector<int> smallestRange(vector<vector<int>>& nums) {
        priority_queue<tuple<int,int,int>,vector<tuple<int,int,int>>,greater<>> pq;
        int n=nums.size();
        int r=INT_MIN;
        for(int i=0;i<n;++i){
            pq.emplace(nums[i][0],i,0);
            r=max(r,nums[i][0]);
        }
        int ans_l=get<0>(pq.top());
        int ans_r=r;
        while(1){
            auto [_,i,j]=pq.top();
            pq.pop();
            if(j+1==nums[i].size()){
                break;
            }
            r=max(r,nums[i][j+1]);
            pq.emplace(nums[i][j+1],i,j+1);
            int l=get<0>(pq.top());
            if(r-l<ans_r-ans_l){
                ans_r=r;
                ans_l=l;
            }

        }
        return {ans_l,ans_r};
    }
};
```





## 逐个语法讲解

### 1. `tuple<int, int, int>`

`tuple` 是 C++11 引入的**固定大小的异类值集合**，可以同时存储多个不同类型的值。这里每个 `tuple` 存三样东西：

| 位置             | 存储内容 | 含义                             |
| ---------------- | -------- | -------------------------------- |
| 第0个 (`get<0>`) | `x`      | 当前元素的值                     |
| 第1个 (`get<1>`) | `i`      | 它来自第几个列表                 |
| 第2个 (`get<2>`) | `j`      | 它是该列表中的第几个元素（下标） |

用 `tuple` 的好处是**不需要单独定义结构体**，一行就能把三种信息绑在一起。

---

### 2. `priority_queue<tuple<int,int,int>, vector<tuple<int,int,int>>, greater<>>`

这是**小顶堆**的完整声明。`priority_queue` 默认是**大顶堆**（最大的在堆顶），用 `greater<>` 可以反转比较方向，变成**小顶堆**。

三个模板参数的含义：

| 参数  | 值                           | 含义                                 |
| ----- | ---------------------------- | ------------------------------------ |
| 第1个 | `tuple<int,int,int>`         | 堆中元素的类型                       |
| 第2个 | `vector<tuple<int,int,int>>` | 底层容器（固定写法）                 |
| 第3个 | `greater<>`                  | 比较器，`greater` 使堆顶为**最小值** |

`tuple` 的比较规则是从第0个元素开始**逐位比较**。当多个元素值相同时，自动按列表编号 `i` 和下标 `j` 决定顺序。

---

### 3. `emplace`（而非 `push`）

```cpp
pq.emplace(nums[i][0], i, 0);
```

`emplace` 是 C++11 引入的**原地构造**函数，它直接在容器内部构造对象，**避免了先创建临时对象再拷贝的开销**。参数直接传给 `tuple` 的构造函数，等价于：

```cpp
pq.push(tuple<int,int,int>(nums[i][0], i, 0));  // 多一次临时对象 + 拷贝
```

---

### 4. `get<0>(pq.top())`

`pq.top()` 返回堆顶元素的**常量引用**。`get<0>()` 是专门用来提取 `tuple` 或 `pair` 中第 `i` 个元素的函数模板。

| 写法               | 作用                                |
| ------------------ | ----------------------------------- |
| `get<0>(pq.top())` | 提取堆顶的第 0 个字段（即元素值）   |
| `get<1>(pq.top())` | 提取堆顶的第 1 个字段（即列表编号） |
| `get<2>(pq.top())` | 提取堆顶的第 2 个字段（即元素下标） |

如果用 `pair<int, int>` 的话，等价的写法是 `pq.top().first`，但 `get<0>` 是适用于 `pair` 和 `tuple` 的**通用写法**。

---

### 5. `auto [_, i, j] = pq.top();`（结构化绑定）

这是 **C++17 的结构化绑定**，可以**一行把元组/结构体的多个字段拆到不同变量里**。

```cpp
auto [_, i, j] = pq.top();
```

等价于：

```cpp
auto tmp = pq.top();
int value = get<0>(tmp);  // 用 _ 表示"这个值我不需要"
int i     = get<1>(tmp);
int j     = get<2>(tmp);
```

`_` 是一个合法的变量名，这里用作**占位符**表示“我不关心这个值”。如果用 `get<0>` 去取值但不使用，会有“变量未使用”的警告；用 `_` 可以干净地告诉编译器“我是故意的”。

`_` 作为“不用变量”的占位符，在 C++26 中会更加标准化，但在 C++17 中就已经可以这样使用。
