# 方法一：差分+前缀和

## 解题思路和过程

因为 `1 <= nums[i] <= limit <= 105` ，所以我们可以知道sum=nums[ i ]+nums[ i-1-i ]的范围是不会超过limit * 2的，同时也还是一定大于等于2的，在知道这一前提后，我们***设target是在最少操作数是对应的使数组互补的时的sum，再看要使数组中的一对数据( a , b )互补的操作次数的大小,可以分为三种情况***：

1. **操作0次，即sum在未操作前就已经等于target了**
2. 操作1次，即通过修改一次数据就能使sum=target，这里我们进一步分析关于target此时对应的范围，加入我们修改a变为x，因为1<=x<=limit ,那就有此时1+b<=sum<=limit+b，即也就是**1+b<=target<=limit+b**，同样加入我们修改的是b，同理可得**1+a<=target<=limit+b，那么关于target更准确的范围就是min(a,b)+1<=target<=max(a,b)+limit**，**换句话来说，只要target在这个范围内，使数组中对应的一对数互补的操作数就是1**
   - 这里补充一点，因为在[min(a,b)+1,max(a,b)+limit]这个范围之间，因为a+b>=min(a+b)+1,且a+b<=max(a+b)+limit，所以a+b是绝对属于这个范围的，那么在通过范围来判断操作次数时不要重复计算

1. 操作2次，毫无疑问，**只要不处于上面两个范围，那么他的操作数就是2**

在得出这一点之后，问题的解决也就不是很难了，根据上面的条件我们直接对操作数据进行统计了，最简单的方法就是直接遍历所有可能的target，然后按照上面的条件计算操作数，**不断用ans更新最小值即可，这也是所谓的暴力统计的方法，不过很可惜这个方法的时间复杂度使O(limit*n)相当于O(n^2)的，在这个题上是超时了的**，这个方法的代码实现贴到本方法的最后，在这我们不在深究

观察上面的三种情况，他们在上面暴力的方法中是通过遍历所有的target来实现的，这里我们转换想法，我们是**可以直接通过给定的范围，直接对这个范围该有的操作数进行计算**呢，毫无疑问当然可以，我们可以直接设置一个数组diff（2*limit+2，0），用diff[ i ]表示target==i时对应的操作数，然后在这个区间找最小值即可，这个算法我们可以发现与上面的时间复杂度几乎相同，因为我们将时间花费到，对一个范围内的数据加同一个值的操作上面了，也是因为这些**值相同我们才可以进一步优化，用差分数组对一个范围内的数据加减同一个数**，这就大大节省我们时间开销了

因为要计算操作两次时，我们没有具体的范围可以直接使用，所以我们可以假设每组数据对操作数ans的贡献都是2，然后遇到在那个范围内的数就-1，再遇到sum==target就在-1（这种情况包括在前面的范围内，不要重复-1）



## 复杂度

- 时间复杂度:O($$n+limit$$)
- 空间复杂度:O($$limit$$)

## Code

    int minMoves(vector<int>& nums, int limit) {

        int ans=INT_MAX;
        for(int i=0;i<n/2;++i){
            int sum=nums[i]+nums[n-1-i];
            diff[sum]+=1;
            diff[sum+1]-=1;
            diff[min(nums[i],nums[n-1-i])+1]+=1;
            diff[max(nums[i],nums[n-i-1])+limit+1]-=1;    
        }
        for(int i=2;i<=2*limit;++i){
            diff[i]+=diff[i-1];
            ans=min(ans,n+diff[i]);
        }

        return ans;

    }
};
    }
};

class Solution {
public:
    int minMoves(vector<int>& nums, int limit) {
        int n=nums.size();
        int ans=INT_MAX;
        for(int i=2;i<=2*limit;++i){
            int res=0;
            for(int j=0;j<n/2;++j){
                int sum=nums[j]+nums[n-1-j];
                if(sum==i){
                    continue;
                }
                else if(i>=min(nums[j],nums[n-1-j])+1 &&
                i<=max(nums[j],nums[n-j-1])+limit){
                    res++;
                }else {
                    res+=2;
                }
            }
            ans=min(ans,res);
        }

        return ans;

    }
};
```

# 方法二：二分查找

## 解题思路和过程

通过上面的分析可得，对于一对数据使其满足互补操作的操作数可以分为三种情况：即0，1，2，见前面我们已经知道了，可以通过范围判断操作数的次数

不知你是否还记得起初的暴力解法，因为暴力解法依次对每一个target对应的操作数进行求解而超时，那么我们是否能在原本暴力解法的基础上进行优化，使其时间复杂度变小呢，我们可以看出超时的核心代码是：（每一个target都要求出一个对应的操作数，不然是无法至知其是最大的，同样前面差分的算法也没有试图简化这一步）

这一部分因为要对nums遍历而使时间复杂度来到了O(n),这段代码的作用是求出在target确定的情况下，nums数组要想互补的操作次数，所以我们要在操作次数的方面分析，也就是上面的三种情况，因为要求操作1次的次数，就避免不了要遍历nums，所以我们是否可以变换方法，不去统计操作1次的次数呢，在上面的方法中我们将diff的值初始化为2，就避免了我们统计操作次数为1的情况，这里我们同样想法的将diff初始为1（因为这题没有用到diff，所以这句话可以理解为，每组数在开始时就默认操作数为1（可以在遇到原本操作数为0的一组数时-1，在遇到原本操作数为2的一组数时+1）），这样避免了统计操作数为1的情况，在分析，统计操作数为零的情况，只需要谋组数sum出现的次数，我们可以在开始前用hashmap统计好，这样就可以直接查询了，（这个操作复杂度为O(1)）,再然后就是原本操作数为2的情况，通过上面的分析只要不在那个范围内，其操作数就为2

因为已知target，我们只需要统计min(a,b)+1>target，max(a,b)+limit<target,的个数就行（target出界了），那么如何计算呢，我们开两个2数组分别存储min(a,b),max(a,b)，然后排序，用表直接计算，但是在这两个数组上面查找时的时间复杂度还是O(n)啊，这是关键就来了，因为这两个数组已经有序了，所以我们可以用二分查找，那他的时间复杂度将降到了O($$longn$$),我们做了那么多事先操作才将复杂度降了这么多，虽然不如差分，但已经足够了



## 复杂度

- 时间复杂度:O($$limit*logn$$)
- 空间复杂度:O($$n$$)

## Code 

```
class Solution {
public:
    int minMoves(vector<int>& nums, int limit) {
        int n=nums.size();
        int ans=INT_MAX;
        unordered_map<int,int> mp;
        vector<int> minval;
        vector<int> maxval;
        for(int i=0;i<n/2;++i){
            if(mp.count(nums[i]+nums[n-1-i])){
                mp[nums[i]+nums[n-1-i]]++;
            }else{
                mp[nums[i]+nums[n-1-i]]=1;
            }
            minval.push_back(min(nums[i],nums[n-1-i]));
            maxval.push_back(max(nums[i],nums[n-1-i]));
        }     
        sort(minval.begin(),minval.end());
        sort(maxval.begin(),maxval.end());
        auto under=[&](int a)-> int{
            int l=0;
            int r=n/2-1;
            int mid;
            while(l<=r){
                mid=(l+r)/2;
                if(minval[mid]+1>a){
                    r=mid-1;
                }else{
                    l=mid+1;
                }
            }
            return n/2-l; 
        };
        auto over=[&](int a)-> int{
            int l=0;
            int r=n/2-1;
            int mid;
            while(l<=r){
                mid=(l+r)/2;
                if(maxval[mid]+limit<a){
                    l=mid+1;
                }else{
                    r=mid-1;
                }
            }
            return l; 
        };
        for(int i=2;i<=2*limit;++i){
            int res=n/2;
            res-=mp[i];
            res+=under(i);
            res+=over(i);
            ans=min(ans,res);
        }
        return ans;
    }
};
```

这里我手搓的二分，也可以直接使用vector的lower_bound

注意，计算原本操作数为2时，可能会有疑问，一组数会不会被重复计数呢？因为target是要出界才能被计数的，同一组数最多只有一个满足条件，因为界限是开箱两端的，同一个target不可能同时满足两个范围，具体如下：
可以用反证法证明，不会存在：
假如 a>c-1 且 b<c-limit，那么 c<a+1 且 c>b+limit，那么 a+1 > b+limit，而我们是假设 a<=b，且预设条件是 1<=limit 的，因此根据预设条件 a+1<=b+limit 的，和我们的结论矛盾，因此不会存在 (a,b)对同时满足，因此可以放心计算。

附上lower_bound的写法：

class Solution {
public:
    int minMoves(vector<int>& nums, int limit) {
        int n=nums.size();
        int ans=INT_MAX;
        unordered_map<int,int> mp;
        vector<int> minval;
        vector<int> maxval;
        for(int i=0;i<n/2;++i){
            if(mp.count(nums[i]+nums[n-1-i])){
                mp[nums[i]+nums[n-1-i]]++;
            }else{
                mp[nums[i]+nums[n-1-i]]=1;
            }
            minval.push_back(min(nums[i],nums[n-1-i]));
            maxval.push_back(max(nums[i],nums[n-1-i]));
        }     
        sort(minval.begin(),minval.end());
        sort(maxval.begin(),maxval.end());
            
        for(int i=2;i<=2*limit;++i){
            int res=n/2;
            res-=mp[i];
            res+=n/2-(lower_bound(minval.begin(),minval.end(),i)
            -minval.begin());
            res+=(lower_bound(maxval.begin(),maxval.end(),i-limit)-maxval.begin());
            ans=min(ans,res);
        }
        return ans;
    }
};
```
