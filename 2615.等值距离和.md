#方法二：分组+前缀和

##解题思路和过程

由上面的暴力算法思路：用两层循环遍历数组，当值相同而坐标不同时，计算坐标差的绝对值即可，而用两层循环来找相同的数太浪费了，那么我们是可以将数组处理一下，让他们放在一起即分组或者排序
有题目可知我们只需要**找到一组相同的数字，计算它们的坐标差的绝对值之和即可**，为了方便分组和方便查找，我们可以用hashmap的不重复的性质，将其相同的数分在同一组，**（hashmap的值是可以为数组的，创建的方式如下）

> unordered_map<int ,vector<long long>> mp;

在分过组之后我们只需要进行坐标的差的绝对值求和即可:那么问题就又显现了，我们发现**要求坐标差的绝对值之和，与相同值的每个坐标都有关系**（如果这里用暴力直接求，时间复杂度甚至可以达到O(n^2)，复杂度完全没有降下来)，给了求这个距离和，我们通过观察可以将其分为：**左边部分与右边部分**分别进行求解再相加
- 以[0,2,3]为例，我们要求2的距离和，即是|2-0|+|3-2|=3，
- 以[0 4 6 8]为例，要求6的距离和，即为|6-0|+|6-4|+|8-6|=10

> 为了批量求距离和，我们可以观察到
1. 左边距离和 = (当前下标 - 左1) + (当前下标 - 左2) + … + (当前下标 - 左m)
假设左边有 m 个下标，那么上面的式子可以写成：
(当前下标 * m) - (左边所有下标之和)
2. 同样，右边的距离和我们也可以得到：
右边距离和 = (右1 - 当前下标) + (右2 - 当前下标) + … + (右k - 当前下标)
假设右边有 k 个下标，式子可化为：
(右边所有下标之和) - (当前下标 * k)
**这也就可以看出我们计算距离和要用的知识了：所有下标之和即可用前缀和来求取**

计算出距离和之后，我们也就只有输出的问题要解决了，因为hashmap中键是原数组值的大小，而值中存储的是对应元素的坐标，故我们可以这个坐标来存储我们的ans

##复杂度

- 时间复杂度:O(n)
- 空间复杂度:O(n)

##Code

```
class Solution {
public:
    vector<long long> distance(vector<int>& nums) {
        unordered_map<int ,vector<long long>> mp;
        int n=nums.size();
        for(int i=0;i<n;++i){
            mp[nums[i]].push_back(i);
        }

        vector<long long> ans(n,0);
        for(auto [key,val]:mp){
            int k=mp[key].size();

            vector<long long> d(k+1,0);
            for(int i=0;i<k;++i){
                d[i+1]=val[i]+d[i];
            }

            for(int i=0;i<k;++i){
                long long left=i*val[i]-d[i+1];
                long long right=(d[k]-d[i])-(k-i-1)*val[i];
                ans[val[i]]=left+right;
            }
        }
        return ans;
    }
};
```
