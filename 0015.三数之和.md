# 方法一：双指针

## 解题思路

因为这道题我们同时面对三个数，倘若我们用暴力做法，直接开三个指针，把每一种情况都比枚举出来时间复杂度来到$O(n^3)$，毫无疑问这太大了，观察数组是无序的，在没有规律可言的情况情况下（题目越短越是麻烦）我们除了枚举也不会有其他办法了，为避免这种情况我们的第一步肯定是排序

**为了避免同时操作和三个指针的复杂情况我们采取固定一个数**，而**同时操作另外两个数的办法**，在原数组排序后的基础上，用循环把每个数依次作为固定值，这时问题就转化为了：**两数之和最接近`0-固定值`的问题**，这个问题我们可以**用双指针来解决**，即用数组两端的双指针分别作为我们要找的另外两个值，这时会有下面的情况：

如果二者和偏大，**这时我们就要让右指针左移，直到我们发现二者的和小于`0-固定值`了，这时我们发现，right右边的值是肯定不会成为我们需要的值，因为他们加最小值都大于`0-固定值`，然后接着如果我们想要等于`0-固定值`，我们就要使二者的和变大，也就是左指针右移（右指针右边的已经淘汰），直到二者和再次大于`0-固定值`，此时类似的left左边的也已经被淘汰，因为他们加上数组可能值的最大值后还是小于`0-固定值`**，以此类推，我们**一定会遇到让二者和最接近`0-固定值`的情况**

然后通过不断改变固定值就可以所有情况下和会等于0的答案了

## 实现细节

1. 遇到三者指针坐标相等的情况可以跳过该次
2. 在等于0填入答案时，同时缩小左右指针，因为只缩小一边，只有相同的值才能再次令和为`0-固定值`，我们不要相同的值，故缩小时我们还要搭配while一直缩小到不相同的值或者两指针相遇为止
3. 固定值以及做右指针没有顺序要求，也就代表我们没必要每次都让left从0出发，可以让他从固定值的右边出发，（同时这也是**为了保证不出现相同的解**）
4. 相同的固定值我们只需要计算一遍，没必要重复计算
5. 因为三者坐标相同，固定值到n-2的位置就可以结束了，后面是left以及right的位置a

## 复杂度

- 时间复杂度:$O(n^2)$，严谨的来说是$O(n^2)+O(n \log n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        int n = nums.size();
        vector<vector<int>> ans;
        for (int i = 0; i < n; ++i) {
            int left = i + 1;
            int right = n - 1;
            if(i>0&&nums[i]==nums[i-1])continue;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    ans.push_back({nums[i], nums[left], nums[right]});
                    left++;
                    right--;
                    while(nums[left]==nums[left-1]&&left<right)left++;
                    while(nums[right]==nums[right+1]&&left<right)right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return ans;
    }
};
```



## 优化

### 优化细节

1. 因为数组是有序的，所以他倘若固定值以及他后边的两位的他们的和已经大于target后面就没必要比较了，差值只会越来越大
2. 类似的，固定值加上数组最大的两位还是小于target，那么小于这两位的也就没必要计算了

```C++
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        int n = nums.size();
        vector<vector<int>> ans;
        for (int i = 0; i < n-2; ++i) {
            int left = i + 1;
            int right = n - 1;
            if(i>0&&nums[i]==nums[i-1])continue;
            if(nums[i]+nums[n-1]+nums[n-2]<0)continue;
            if(nums[i]+nums[i+1]+nums[i+2]>0)break;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    ans.push_back({nums[i], nums[left], nums[right]});
                    left++;
                    right--;
                    while(nums[left]==nums[left-1]&&left<right)left++;
                    while(nums[right]==nums[right+1]&&left<right)right--;
                } else if (sum < 0) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        return ans;
    }
};
```
