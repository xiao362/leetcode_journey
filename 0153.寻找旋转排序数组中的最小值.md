# 方法一：二分

## 解题思路和过程

如果题目不要求复杂度，我们倒是可以直接遍历一遍就能直接得出结果，这样的时间复杂度是$O(n)$,但是题目要求了时间复杂度是$O(log n)$，那我们也就只能优化原来的算法，或者使用其他的算法了，看到要求的时间复杂度$O(logn)$，在看到数组元素分为两部分有序，那最常见的办法就是二分，如果是二分的话，check函数要咋写呢，也就是如果了，l，r要改变范围的条件是什么呢，我们可以观察到无非就3种情况：

1. nums[ mid-1 ],nums[ mid ],nums[ mid+1 ],这三个数是不完全有序的，那么直接取三者中的最小值即可，同时因为他们不完全有序就代表他们是两部分有序数组的交界，那nums[ mid-1 ] 肯定大于后面两个，那就直接取后面两个的最小值即可，这是可以直接返回，这是算法的终止条件
2. 他们三个nums[ mid-1 ],nums[ mid ],nums[ mid+1 ]基本有序，这又可以分为两种情况：
   - mid-1 到r 数组完全有序的，这时numd[ r ]>nums[ mid+1 ]，为了找最小值，将右边界缩小
   - 如果mid-1 到 r不完全有序，那就说明mid+1部分肯定在较大有序数组部分，这时直接缩小左边界

除了这三种情况外，还需要考虑出界的问题，首先可以在n为1，2时直接返回答案，太短了也不至于用二分，然后遇到边界时，直接返回没出界数的最小值即可，关于为什么用nums[ mid-1 ],nums[ mid ],mid[ mid+1 ],来确定结果，这是因为我们的check是基于顺序的，为了得到最小值，我们最好使用一个小区间来确定，就像6，7，1，2，3，4，5这时如果我们直接更该左边界，我们是不知道要直接更改多少的

## 复杂度

- 时间复杂度:$O(log n)$
- 空间复杂度:$O(1)$,不算题目中给出的nums

## Code

```c++
class Solution {
public:
    int findMin(vector<int>& nums) {
        int n=nums.size();
        if(n==1)return nums[0];
        else if(n==2) return min(nums[0],nums[1]);
        int l=0;
        int r=n-1;
        while(l<=r){
            int mid=(l+r)>>1;
            if(mid+1==n){
                if(mid-1>=0) return min(nums[mid-1],nums[mid]);
                else return nums[mid];
            }else if(mid-1<0){
                if(mid+1!=n)return min(nums[mid],nums[mid+1]);
                else return nums[mid];
            }else if(nums[mid-1]<nums[mid+1]){
                if(nums[r]<nums[mid+1]){
                    l=mid+1;
                }else{
                    r=mid-1;
                }
            }else{
                return min(nums[mid],nums[mid+1]);
            }
        }

        return nums[l];
    }
};
```

## 解题思路和过程

通过前面的分析我们可以知道，这个数组被分成了两个相对有序的部分（**左右两部分都是递增的**），在这的基础上，我们要做的就是找到两各数组的交界处，右边数组的第一个值，为此我们可以使用二分的特点，每次舍弃掉无用的一半，这样就可以实现$O(log n)$的时间复杂度，这里我们记 l , r 分别为左右边界，mid为中间值，这时我们会遇到两种情况：

1. **nums[ mid ]<nums[ r ]，这也就说明mid到r中间的距离内是不会出现最小值的，直接舍弃即令r=mid，这里不用能用mid-1，因为mid-1位置是否是最小值还不确定**
2. **nums[ mid ]>nums[ r ]，那就说明 l 到mid是不会出现最小值的，且因为r对应的值其小mid位置也绝对不是最小值，即令 l =mid+1**

> ***注意：因为我们没有处理nums[mid]==nums[r]的情况，此时如果二者相等，则会陷入死循环，为避免这种情况直接再mid=r时返回即可***

 ## 复杂度

- 时间复杂度:$O(\log n)$
- 空间复杂度:$O(1)$

## Code

```c++
class Solution {
public:
    int findMin(vector<int>& nums) {
        int n=nums.size();
        int l=0;
        int r=n-1;
        while(l<r){
            int mid=(l+r)>>1;
            if(nums[mid]<nums[r]){
                r=mid;
            }else if(nums[mid]>nums[r]){
                l=mid+1;
            }
        }
        return nums[l];
    }
};
```
