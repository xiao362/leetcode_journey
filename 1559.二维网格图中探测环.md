#方法一：dfs

##解题思路和过程

由题目提供的图我们可以轻易的想出使用bfs或者dfs的，而为了成环，我们可以想到的就是当他再次遇到他已经走过的路时，便注定有一个环的产生，所以最直接的想法就是使用dfs（bfs也可以用，甚至更简单），**从一点出发，当他遇到不是自己的父节点，且是曾经走过的路的时候，就说明有环产生了，直接结束程序即可**，题目没有要求统计环的个数

> 关于递归是否需要手动维护栈的简单判断条件就是看其深度是否超过一万层，这道题倒是可以直接递归，这道题是可以直接递归的

##复杂度

- 时间复杂度:O(n*m)
- 空间复杂度:O(n*m)

#Code

```
    class Solution {
    public:
        const int dx[4]={1,0,-1,0};
        const int dy[4]={0,-1,0,1};
        struct position{
            int x;
            int y;
            int fx;
            int fy;
        };
        bool dfs(int i,int j,vector<vector<bool>>& vis,int n,int m,vector<vector<char>>& grid){

            stack<position> sta;
            sta.push({i,j,-1,-1});
            vis[i][j]=1;
                while(!sta.empty()){
                int x=sta.top().x;
                int y=sta.top().y;
                int fx=sta.top().fx;
                int fy=sta.top().fy;
                sta.pop();
                    for(int k=0;k<4;++k){  
                        int nx=x+dx[k];
                        int ny=y+dy[k];
                        if(nx>=0&&nx<n&&ny>=0&&ny<m //保证没出界
                        &&vis[nx][ny]==1  //确保是走过的路
                        &&nx!=fx&&ny!=fy&&grid[nx][ny]==grid[x][y]){ //保证不不是父节点，并且可以走 
                            return 1;
                        }
                        if(nx>=0&&nx<n&&ny>=0&&ny<m&&!vis[nx][ny] //保证没有走过
                        &&grid[nx][ny]==grid[x][y]){
                            sta.push({nx,ny,x,y});
                            vis[nx][ny]=1;
                        }  
                        
                    }
                }
                return 0;
        }
        bool containsCycle(vector<vector<char>>& grid) {
            int n=grid.size();
            int m=grid[0].size();
            vector<vector<bool>> vis(n,vector<bool> (m,0));
            
            for(int i=0;i<n;++i){
                for(int j=0;j<m;++j){
                    if(!vis[i][j]){
                        if(dfs(i,j,vis,n,m,grid))return 1;//子要能成环直接返回
                    }
                }
            }

            return 0;
        }
    };
```

因为将函数放在外边的原因，参数可能较为复杂，还有判断条件也较为复杂

#方法二：并查集

##解题思路和过程

同样是上面的判断题条件：这次使用并查集，并**将二维的并查集转化为一维的并查集来存储：id=i*m+j;**
也因此，当我们在遍历的时候是一行一行的遍历，为了保证有环时能够相遇，又因为并查集是直接找到当前位置的根的(可以理解为他的父亲祖宗节点，所以我们可以省略上边和左边这两个方向，**只遍历右边和下边这两个方向**)，当**遍历时发现有节点和自己具有相同的祖宗时，即代表成环了**

> 相当是，要成为一个环，上边的边和下边的边要有两条线将他们连起来(左右各一条)，而遍历完最上面一行时，下一行的左右两边的线就已经遍历好了，即当他遇到第二行的左边一条边的时候，右边的也遍历好具有和他相同的祖宗了，这时，如果左边的一条边可以想右（横向）遍历时，当他能遇到右边的那个点时，就证明下面的一条线就连起来了，即闭环了

##复杂度

- 时间复杂度:O(n*m)
- 空间复杂度:O(n*m)

##Code

```
class Solution {
public:
    bool containsCycle(vector<vector<char>>& grid) {
        int n=grid.size();
        int m=grid[0].size();
        vector<int> parent(n*m);
        for(int i=0;i<n*m;++i){
            parent[i]=i;
        }

        function <int(int)>  find = [&](int a){
            if(parent[a]==a) return a;
            return parent[a]=find(parent[a]);
        };

        auto checkAndMerge = [&](int a,int b)-> bool{
            int ra=find(a);
            int rb=find(b);
            if(ra==rb)return 1;
            parent[rb]=ra;
            return 0;
        };

        for(int i=0;i<n;++i){
            for(int j=0;j<m;++j){
                int id=i*m+j;
                if(j+1<m&&grid[i][j]==grid[i][j+1]){
                    if(checkAndMerge(id,id+1))   return 1;            
                }    
                if(i+1<n&&grid[i][j]==grid[i+1][j]){
                    if(checkAndMerge(id,id+m))return 1;
                }
            }
        }

        return 0;
    }
};
```
