# 方法一：排序+滑动窗口

## 解题思路和过程

通过题意我们要求得使数组平衡的最小转移数目，平衡得概念：如果一个数组的 **最大** 元素的值 **至多** 是其 **最小** 元素的 `k` 倍，则该数组被称为是 **平衡** 的。如果我们观察给出的示例数据，我们在模拟这个过程是我们倘若通过记录每个区间的最小值，来逐一判断某个值是否要加进去，在遇到更小的值后，原来加进去的只有是否要取出来我们完全无法判断，倒是可以通过从头在遍历一遍看那些可以加，那些不能进，这样做的复杂度是$O(n^2)$，如果这样就结束的话，复杂度到还能接受(**我能接受，题目好像接受不了，^ \/ ^**，毕竟；leetcode骗点数据也没啥用)，但是我们忽略**并否是整个数组的最小值维护的范围就是最优的**,所以可能还要再加上一层，那就是天文数字了，我都接受不了了，所以这无疑是不行的

那么我们就要再这基础上优化，我们上面复杂度高的主要原因是要维持最小值且以及某些数是否应该出，我们呢是否有办法同时做到这一点且比哦正时间复杂度不会超了呢，要用到最小值，还要使剩余数组中的数**满足`max <= min * k`**，既然跟大小有关系那我们直接**排序**就好了，排过序后，用left为坐标维护left到right其间的值都满足上面的条件，这就是**不定长滑动窗口**的问题了，我们用双指针来维护区间值，**一旦`nums[right]>nums[left]*k`,就让left右移即可直到符合要求**，最后我们**可以通过求出这个区间的最大值，反向的求出删掉的元素个数**

> 注意：nums[left]*k可能会溢出，可以把这个数据强转为long long

## 复杂度

- 时间复杂度:$O(nlogn)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int minRemoval(vector<int>& nums, int k) {
        int n=nums.size();
        sort(nums.begin(),nums.end());
        int left=0;
        int len=0;
        for(int i=0;i<n;++i){
            while(nums[i]>(long long)nums[left]*k)left++;
            len=max(len,i-left+1);
        }
        return n-len;
    }
};
```
