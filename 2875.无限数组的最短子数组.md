# 方法一：滑动窗口

下面会用的函数

`std::reduce` 的核心作用可以用一句话概括：**对一个范围内的元素进行任意顺序的累加（或自定义运算），最终得到一个汇总结果。**

```
long long total = std::reduce(v.begin(), v.end(), 0LL);
```

这行代码遍历 `v` 中的每一个元素，把它们全部加起来，初始值设为 `0`，最终结果类型由初始值 `0LL` 决定（这里是 `long long`）。相当于 `(((0 + v[0]) + v[1]) + ... + v[n-1])`，但不保证括号的结合顺序。

`accumulate` 严格从左到右加，而 `reduce` 允许编译器打乱顺序来优化（比如利用向量指令或并行）。因此，对于那些不依赖运算顺序的操作（如整数加法、乘法），`reduce` 能提供更好的性能，是现代 C++ 追求零成本抽象的一个体现。

## 解题思路和过程

首先这道题的求和判断的思路很简单，但是关键在于我们要在何时终止循环，因为倘若按照他的永无止境的思路，我们怕是要炸了，我们能想到最简单的的终止条件，应该就是，让再次遇到同样长度的且sum==target的数组时，终止条件，但是对于一些极限数据，**比`nums =[1,1,1]，target =1000000000`时总是避免不了要爆时间**，那我就还要**优化我们的终止条件**

那么我们是否可以换种思路，从爆时间的根源问题出发，**爆时间主要是因为多次的进行运算**，且这一题中，因为数组是无限长的，且是有多个数组拼接成的，且在如上面的极限数据时**会反复计算1+1+1那我们可以直接求出数组nums的元素和total，然后减少重复计算**，下面对一组数据分析`nums =[1,2,3]，target =13`，那就有无限数组[1,2,3，1,2,3，1,2,3，1,2,3，1,2,3，1,2,3，1,2,3]，要找到最优解[1,2,3，1,2,3，1],会重复计算1+2+3，两次，我们可以直接**用target%total舍去那些重复的和total，剩下的则在nums找，准确的说是在长度为2*n的nums里找**，用滑动窗口

关于这个方法可能会有一些疑问，**left，right不是在边界开始的话，那我们找到剩下的不久和我们除去省略计算的数据有间隔了吗**？，不**他还是连续的，就像[1,2,3，1,2,3，1]中，[1,2,3,1,2,3]也可以换成[2,3,1,2,3,1]**，即我们**省略计算的部分是可以和我们left或者right接上**的

第二个就是为啥范围是0~2*n？数组nums经过拼接的特性，不然就无法**联系nums的开头和结尾**(避免漏解)，并且**剩下的数是不会超过total的**

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int minSizeSubarray(vector<int>& nums, int target) {
        long long total=reduce(nums.begin(),nums.end(),0LL);
        int ans=INT_MAX;
        int left=0;
        int sum=0;
        int n=nums.size();
        for(int i=0;i<n*2;++i){
            sum+=nums[i%n];
            while(sum>target%total){
                sum-=nums[left++%n];
            }
            if(sum==target%total){
                ans=min(ans,i-left+1);
            }
        }
        return ans==INT_MAX?-1:ans+target/total*n;
    }
};
```
