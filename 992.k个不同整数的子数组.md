# 方法一：滑动窗口

恰好型滑动窗口

## 解题思路

因为我们的研究对象是子数组，也就是一段连续的区间，所以我们一般用滑动窗口来解决这个问题，但是常规的滑动窗口一般解决的是有边界不动，用左边界来维护区间长度的办法来统计，子数组个数或者其他的信息，但是根据数据`[1,2], [2,1],[1,2,1]`这种方法是会漏解的，所以我们要换种思路

我们可以通过统计不同元素个数大于等于k以及k+1的个数用而二者的差来计算恰好等于k的子数组的个数

## 实现细节

1. 统计不同元素个数大于等于k的子数组的个数时，要用`ans+=left` 来更新个数，含义就是维护左边界使区间一直处于刚好曼珠条件的状态，这样就左边界到0的个数来计算大于等于k的子数组个数
2. 如果不想写两遍上面的过程可以把上面的功能写成一个函数，但时间上可能会稍慢

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(k)$

## Code

```C++
class Solution {
public:
    int solve(vector<int>& nums, int k){
        int n=nums.size();
        int ans=0;
        int left=0;
        int cnt=0;
        unordered_map<int,int> mp;
        for(int i=0;i<n;++i){
            mp[nums[i]]++;
            if(mp[nums[i]]==1)cnt++;
            while(cnt==k){
                mp[nums[left]]--;
                if(mp[nums[left]]==0)cnt--;
                left++;
            }
            ans+=left;
        }
        return ans;
    }

    int subarraysWithKDistinct(vector<int>& nums, int k) {
        return solve(nums,k)-solve(nums,k+1);
    }
};
```
