# 方法一：bfs广度优先算法+记忆化

## 解题思路和过程

依据题意，然后通过样例数据，先来大致模拟这整个的过程，首先我们从起点出发，这时**我们的下一步会面对三种方向：**

1. `i-1`，当然起点的这个方向肯定是不成立的
2. `i+1`
3. `j` 需满足：`arr[i] == arr[j]` 且 `i != j`

> 注意以上三点都要保证在数组的范围之内

然后他要求我们返回到达终点的最小操作次数，这就是跳跃次数，对于这种**找最佳路径问题我们常用的两种办法：dfs及bfs**，因为数据较大，**倘若使用dfs，递归深度可能会过深**（为防止栈崩溃，可能还要自己手搓栈空间，稍微有点麻烦，最重要的是：我们要统计最小操作次数，而**dfs要走到底才能统计出来，还用不了记忆化，最后肯定会爆空间和时间**），相比之下**用bfs配合记忆化，并在到达终点时直接结束程序，可以避免重复访问，大大节约了时间**

在确定使用bfs后，我们可以着手代码实现了，首先我们可以将**符合`j` 需满足：`arr[i] == arr[j]` 且 `i != j`条件的坐标放在一起**，毕竟另外两个坐标通过加减可以直接算出来，而这个不能，我们可以实现对数据处理一遍，**用hashmap配和vector将arr值相同的坐标放在一起，加上hash易查找的特性不会增加我们bfs的时间复杂度**，然后我们就要开队列，因为**要统计最小操作次数，故我们可以开两个队列，将每一步分开操作**，然后就是开**记忆化数组vis**，这防止我们重复访问某一点，用**ans记录步数**，比较关键的步骤就是将下一步入队，这时我们要保证两个条件：

1. 不越界，idx>=0&&idx<n
2. 没访问过，vis==0

> 注意ans的位置要保证每一步执行完后再++

## 复杂度

- 时间复杂度:$O(n)$,事先处理数据也是$O(n)$
- 空间复杂度:$O(n)$，记忆化数组以及队列最大空间也都是$O(n)$

## Code

```C++
class Solution {
public:
    int minJumps(vector<int>& arr) {
        int n=arr.size();
        int ans=0;
        queue<int> q;
        vector<int> vis(n,0);
        unordered_map<int,vector<int>> mp;
        for(int i=0;i<n;++i){
            mp[arr[i]].push_back(i);
        }
        q.push(0);
        vis[0]=1;
        while(!q.empty()){
            queue<int> q2;
            while(!q.empty()){
                int idx=q.front();
                if(idx==n-1)return ans;
                q.pop();
                if(idx-1<n&&idx-1>=0&&vis[idx-1]==0){
                    q2.push(idx-1);
                    vis[idx-1]=1;
                }
                if(idx+1>=0&&idx+1<n&&vis[idx+1]==0){
                    q2.push(idx+1);
                    vis[idx+1]= 1;
                }
                if(!mp[arr[idx]].empty()){
                    for(auto i:mp[arr[idx]]){
                        if(vis[i]==0){
                            q2.push(i);
                            vis[i]=1;
                        }
                    }
                    mp[arr[idx]].clear();//上面没具体说，这里可以直接清空或者放在循环里一个个erase可以避免重复访问
                    //虽然不清空不会导致错误答案，但这样多少也节省时间
                }
            }
            q=q2;
            ans++;
        }
        return ans;
    }
};
```
