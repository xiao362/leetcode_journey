# 方法一：双指针

## 解题思路

这道题最直接解法就是暴力枚举把每一种情况都列举一遍，但是因为时间复杂度为$O(n^2)$，会超时，说以我们要想办法优化这个算法，或者换一种更加高效的算法，模拟观察枚举过程，以示例一为例：

![img](https://aliyun-lc-upload.oss-cn-hangzhou.aliyuncs.com/aliyun-lc-upload/uploads/2018/07/25/question_11.jpg)



```
输入：[1,8,6,2,5,4,8,3,7]
输出：49 
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。
```

枚举过程中：当我们以`0`位置为左边界时，对应的最大面积是，最右边的值因为他的高度是最低的，所以高度确定的情况下距离最远的就是最大的，当我们以`1`位置为左边界时，我们发现最大的面积对应的右坐标还是右边界，紧接着当我们以`2`为左边界时，我们发现最大面积对应的边界还是还是右边界，如此反复我们发现全是右边界。

**难道我们可以直接不考虑中间的其他边界只考虑右边界就行了？这样时间复杂度就是$O(n)$了也太爽了**，肯定不是了，右边界没有什么特殊，只是这个数据凑巧了，假如我们让这份数据反转一下让`0`位置做右边界的话。

以右边`0`位置为起点，对应的最大值就是左边第`2`个边界，然后再以右边`1`位置为起点，最大面积对相应的边界还是左边第`2`个边界，同样以左边`2,3,4`为起点也依旧是左边第`2`个边界，观察这两数据的共同性，就是离他最远的大于等于的边界，从这一点切入：

**我们求面积时是以较低的一边为高的，也就是说如果我们移动较高的一边，只能通过像距离远的位置移动才能使面积变大，如果距离变小的话，无论遇到比之前高多少的边，它的面积都只会减小，遇到低的边就更不用说了，那也就代表中间的全被淘汰了，那么假如我们使用两个指针从两端出发，他的距离就没办法变大，里面的又被排除（与里面的都被遍历一遍无异），所以如果我们想要面积变得更大，就只能通过移动较小的边才有可能增加面积，而移动过后我们又会面临同样的问题：怎么使最大面积变大（不用回退了，我们统计最大面积，刚刚的情况已经考虑完了），那我们只能接着移动较小的一边了，如此反复最大面积就求出来了**

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int maxArea(vector<int>& height) {
        int n=height.size();
        int ans=0;
        int left=0;
        int right=n-1;
        while(left<right){
            if(height[left]<height[right]){
                ans=max(ans,(right-left)*height[left]);
                left++;
            }else{
                ans=max(ans,(right-left)*height[right]);
                right--;
            }
        }
        return ans;
    }
};
```
