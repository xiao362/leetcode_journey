那**如果我们事先不知道任何坐标位置，只是设两个指针在数组的两端，情况是否与上面类似呢，如果二者和偏大，这时我们就要让右指针左移，直到我们发现二者的和小于target了，这时我们发现，right右边的值是肯定不会成为我们需要的值，因为他们加最小值都大于target，然后接着如果我们想要等于target，我们就要使二者的和变大，也就是左指针右移（右指针右边的已经淘汰），直到二者和再次大于targte，此时类似的left左边的也已经被淘汰，因为他们加上数组可能值的最大值后还是小于target，以此类推，我们一定会遇到让二者和等于target的情况**

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int n=numbers.size();
        int left=0;
        int right=n-1;
        while(left<right){
            if(numbers[left]+numbers[right]==target){
                return {left+1,right+1};
            }else if(numbers[left]+numbers[right]>target){
                right--;
            }else{
                left++;
            }
        }
        return {0,0};
    }
};
```
