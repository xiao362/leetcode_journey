# 方法一：bfs

## 解题思路

按照题目我们模拟，跳跃过程，是首先我们从起点出发**，然后在`i + minJump <= j <= min(i + maxJump, s.length - 1)` 的范围内找`s[j] == '0'`的点**，也就是我们可以跳跃的点，这个过程倒是不难我们可以直接暴力实现，但是倘若我们遇到一些极限的数据时，如`minJump=1`并且`maxJump=n/2` s 的值也全为 ‘ 0 ‘ 时，这时候我们用常规的队列写法，这时我们的队列里面已经有前n/2的元素了，在idx变为2，时我们又要从3依次开始遍历查询，虽然可以使用记忆化不让其进队不影响结果但是判断也是耗费了相当一部分时间，这样时间复杂度也就来到了$O(n^2)$

这时我们可以**引进一个变量用来记录我们已经判断过的最远距离last，避免重复判断**

## 实现细节

1. 用**队列存我们可以跳跃的坐标**
2. **`vector<bool> vis(n,0)`用来记忆化避免重复入队**
3. last存我们的判断过的最远距离（last前面的已经判断过），即**`last=max(last,r)`;**
4. 满足条件`!vis[i]&&s[i]=='0'`
5. 返回`vis[n-1]`即可

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n=s.size();
        vector<bool> vis(n,0);
        vis[0]=1;
        int last=0;
        queue<int> q;
        q.push(0);
        while(!q.empty()){
            int idx=q.front();
            q.pop();
            int l=max(idx+minJump,last);
            int r=min(idx+maxJump,n-1);
            last=max(last,r);
            for(int i=l;i<=r;++i){
                if(!vis[i]&&s[i]=='0'){
                    vis[i]=1;
                    q.push(i);
                }
            }
        }        
        return vis[n-1];
    }
};
```

# 方法二：动态规划dp+前缀和

## 解题思路

上面的方法是，通过我们现在的坐标来找出我们所有可以走到的地方，倘若我们换种思路，这次我们**不再往后找而是向前找**，也就是说

倘若我们现在位于起点，这也就**意味着只要在`i + minJump <= j <= min(i + maxJump, s.length - 1)` 区间（这是i能到达的j的所有位置）内，只要`s[j]=='0'`那我们就可以到达这个位置**，换种思路也就是，我们如果在`j`的位置

只要**`max(j-maxJump,0)<=i<=j-minJump`内存在可以跳跃到`i`的位置那我们就可以跳到如今`j`的位置**，（这里就是依照前面的状态来推现在的状态，即就是**动态规划**的思想），但是这里同样面对和上面类似的问题，那就是当遇到极限的数据时我们的时间复杂度会来到$O(n^2)$

上面我们用last来记忆我们走过的最远距离，但是这里我们要实打实的遍历完`max(j-maxJump,0)<=i<=j-minJump`区间的话，难免超时，故我们要优化，不走全程就能知道这个区间内是否有能跳跃的位置，注意到我们判断的是有没有的问题，也就是bool的问题，那么我们**让每一步继承前面的状态（用同前缀和来继承状态**），因为**前缀和是非递减的**，在某一个地方能条约（1）我们直接加上就行，这样我们就可以**通过前缀和是否增加来判断，某个区间是否有（1）**

## 实现细节

1. 用vector<int> f(n,0)，来表示当前位置，是否能够从起点跳跃过来（用bool更合适）
2. vector<int> diff(n+1,0)，来表示前缀和数组
3. 别忘了初始化起点的状态
4. 跳跃的前提是s[i]='0'
5. 边界问题，如果右边界小于0无意义（对应前面方法该位置在`i+minJump`的左边）

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n=s.size();
        vector<int> f(n,0);
        f[0]=1;
        diff[1]=1;
        for(int i=1;i<n;++i){
            if(s[i]=='0'&&i-minJump>=0&&diff[i-minJump+1]>diff[max(i-maxJump,0)])f[i]=1;
            else f[i]=0;
            diff[i+1]=f[i]+diff[i];
        }
        return f[n-1];
    }
};
```

这里可以进一步优化，把f给省略掉

## 优化

最简形式

class Solution {

class Solution {

    bool canReach(string s, int minJump, int maxJump) {
        int n=s.size();
        vector<int> diff(n+1,0);
        diff[1]=1;  
        for(int i=1;i<n;++i){

# 方法三：差分+前缀和

## 解题思路

这个思路大体上与第一种方法相同，第一种方法我们通过记忆化来避免对`i + minJump <= j <= min(i + maxJump, s.length - 1)` 区间的遍历，第二个方法我们通过dp与前缀和搭配反向的判断某一位置是否可以跳跃，这里我再次强调一下可以跳跃的条件

1. 满足在`i + minJump <= j <= min(i + maxJump, s.length - 1)` 区间内
2. `s[j]==‘0’`

第二个条件是我们遍历到某一个点直接查询的，而第一个条件，我们可以使用使用一个数组(初始化为0)，对每个元素对应的区间的元素++来标记，**只要值大于1那就证明前面有个值可以跳到这里，同样我们不能直接对每个元素依次+1，而是利用差分加前缀和的组合直接对某一个区间+1**，而只要满足第二个条件就可以发生跳跃

## 实现细节

1. 用`vector<int> diff(n+1,0)`时注意数组长度是n+1（要包括整个区间）
2. 可以用`int sum_d=0`，来求前面的前缀和，用于判断当前元素是否可以跳跃
3. 差分数组区间+1时后面的坐标要加1（前闭后开）`diff[min(n,i+maxJump+1)]--`
4. 返回时也要判断`s[n-1]=='0'`

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n=s.size();
        vector<int> diff(n+1,0);
        diff[0]=1;
        diff[1]=-1;
        int sum_d=0;
        for(int i=0;i<n;++i){
            char ch=s[i];
            sum_d+=diff[i];
            if(sum_d>0&&s[i]=='0'){
                diff[min(n,i+minJump)]++;
                diff[min(n,i+maxJump+1)]--;
            }
        }
        return sum_d>0&&s[n-1]=='0';
    }
};
```

# 同向双指针

本质与前面bfs相同

## 解题思路

基本与前面bfs的思路大致相同，可以说是换了种写法，这里用right保证不走已经走过的区间元素，并换用vis代替queue可以看成第一个方法的别样实现形式

## 实现细节

1. 用`right`更新走过的最大区间的右边界
2. `vector<bool> vis(n,0)`保存是否满足第一个条件

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(n)$

## Code

```C++
class Solution {
public:
    bool canReach(string s, int minJump, int maxJump) {
        int n=s.size();
        vector<bool> vis(n,0);
        vis[0]=1;
        int left=0;
        int right=0;
        for(;left<n;++left){
            if(vis[left]&&s[left]=='0'){
                for(right=max(right,left+minJump);right<=min(n-1,left+maxJump);++right){
                    vis[right]=1;
                }
            }
        }
        return vis[n-1]&&s[n-1]=='0';
    }
};
```
