# 方法二：二分

**根据情况改变二分的条件，包括终止条件以及范围更新的条件，不要死套模板**

## 解题思路和过程

通过前面的分析我们可以知道，这个数组被分成了两个相对有序的部分（**左右两部分都是递增的**），在这的基础上，我们要做的就是找到两各数组的交界处，右边数组的第一个值，为此我们可以使用二分的特点，每次舍弃掉无用的一半，这样就可以实现$O(log n)$的时间复杂度，这里我们记 l , r 分别为左右边界，mid为中间值，这时我们会遇到三种情况：

1. nums[ mid ]<nums[ r ]，这也就说明mid到r中间的距离内是不会出现最小值的，**直接舍弃即令r=mid，这里不用能用mid-1，因为mid-1位置是否是最小值还不确定**
2. nums[ mid ]>nums[ r ]，那就说明 l 到mid是不会出现最小值的，且**因为r对应的值其小mid位置也绝对不是最小值，即令 l =mid+1**
3. nums[mid]==nums[ r ]**,这时我们就无法判断顺序，也就无法判断要舍弃的空间，**为此我们采用一个巧妙的办法，**r--，因为有mid与原来的值相等倒不会失去最小值，当变到一个原来不等的值时就可以判断顺序，进行二分了**



 ## 复杂度

- 时间复杂度:$O(\log n)$
- 空间复杂度:$O(1)$

## Code

```
class Solution {
public:
    int findMin(vector<int>& nums) {
        int n=nums.size();
        int l=0;
        int r=n-1;
        while(l<=r){
            int mid=(l+r)>>1;
            if(nums[mid]==nums[r]){
                r--;
            }else if(nums[mid]<nums[r]){
                r=mid;
            }else{
                l=mid+1;
            }
        }
        return nums[l];
    }
};
```
