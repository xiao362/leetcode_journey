#方法一：模拟

##解题思路和过程

通过观察我们可以发现，所谓z字变换不过是将一条直线拉弯了，换种说法来说不过是，将原本只有一个方向（水平向右）变成了两个方向(向下，以及向右上)，在抽象点（不把他看成z，而看成w，顶点很近的w，那就可以将其抽象为上下两个方向），即是，当i(控制行数)触碰到两条边界时，就会掉转方向

> 这里要注意一点，结果的顺序是按照行输出的，故我们可以按照行来存储数据，如果要开数组（二维矩阵），代码实现还是有点复杂的，当然开矩阵也是最直观，易想得到的办法

关于代码实现时，我们可以用一个bool变量来控制方向

##复杂度

- 时间复杂度:O(n)
- 空间复杂度:O(n)

##Code

```
class Solution {
public:
    string convert(string s, int numRows) {
        if(numRows==1) return s;
        vector<string> p(numRows);
        bool todown=1;
        int i=0;
        for(char ch:s){
            if(todown){
                p[i]+=ch;
                ++i;
                if(i==numRows-1){
                    todown=0;
                }
            }else{
                p[i]+=ch;
                i--;
                if(i==0){
                    todown=1;
                }
            }
        }

        string ans="";
        for(auto it:p){
            ans+=it;
        }

        return ans;
    }
};
```

##代码优化

> 使代码更加简洁容易，逻辑更清晰

```
class Solution {
public:
    string convert(string s, int numRows) {
        if(numRows==1)return s;
        vector<string> p(min(numRows,(int)s.size()));
        bool goingdown=0;
        int i=0;
        for(char ch:s){
            p[i]+=ch;
            if(i==0||i==numRows-1){
                goingdown=!goingdown;
            }
            i+= goingdown ? 1:-1;
        }
        string ans="";
        for(auto it:p){
            ans+=it;
        }
        return ans;
    }
};
```

#方法二：二维矩阵模拟

> 同样是模拟，结构比较清晰，但这个代码实现起来可能稍显复杂

##解题流程和思路

这个不需要抽象可以直接按照题目进行模拟，但是要注意，所开的二维数组的大小，不能太大，计算列数可能会出错

> 先观察，一个周期内共有几个元素cycleLen=2*numRows-2个，一个周期占了numRows-1列，**最后不足一个周期的要向上取整补成一个周期**：
- 在原本的一个（数+T-1）/T
其中的+T-1就是起到了向上取整的作用

##复杂度

- 时间复杂度:O(n^2)/O(numRows*n)
- 空间复杂度:O(n^2)/O(numRows*n)

##Code

```
class Solution {
public:
    string convert(string s, int numRows) {
        if(numRows==1) return s;
        int n=s.size();
        int cycleLen=2*numRows-2;//一个周期的元素个数
        int T=(n+cycleLen-1)/cycleLen;//周期
        T*=(numRows-1);//列数

        vector<vector<char>> mat(numRows,vector<char> (T,'\0'));
        bool goingdown=1;

        int i=0,j=0;
        for(char ch:s){
            mat[i][j]=ch;
            if(goingdown){
                i++;
            }else{
                i--;
                j++;
            }
            if(i==0||i==numRows-1){
                goingdown=!goingdown;
            }
        }
        string ans;
        for(int i=0;i<numRows;++i){
            for(int j=0;j<T;++j){
                if(mat[i][j]!='\0'){
                    ans+=mat[i][j];
                }
            }
        }
        return ans;
    }
};
```

> 方法一的模拟方法可以看作是矩阵的压缩


#方法三：直接构造（数学规律）

> 这种方法一般很难想到，这需要很强的观察能力，能够观察出数与数之间的关系，因为比较巧妙，所以一般复杂度较低
总之用这种方法最好的途径就是像拉马努金一样<<观察到>>

##解题思路和过程

通过观察我们可以注意到，记行数为i，周期列数为j：

**- 在第一行以及最后一行他们每个元素坐标差都是一个周期t**
**- 除第一行和最后一行外其他行只要是完整周期，其在每一行都会有两个数，一个是与第一行横坐标i相同（二维）或与其先相连（一维），另一个数的坐标始终是j-i **

综上因为有一个数，无论在哪一行都与有关，然后我们就可以开两个循环，来直接构造每一行的字母了

##复杂度

- 时间复杂度:O(n)
- 空间复杂度:O(1)

##Code

```
class Solution {
public:
    string convert(string s, int numRows) {
        int n=s.size();
        if(numRows==1||numRows>=n){
            return s;
        }
        string ans="";
        int t=2*numRows-2;
        for(int i=0;i<numRows;++i){
            for(int j=0;i+j<n;j+=t){
                ans+=s[i+j];
                if(i!=0&&i!=numRows-1&&j-i+t<n){
                    ans+=s[j-i+t];
                }
            }
        }
        return ans;
    }
};
```
