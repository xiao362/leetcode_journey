# 方法一：dp+dfs

学艺不精 才疏学浅 妄自菲薄

## 解题思路和过程

这道题还是相当有难度的，单单是读懂题意就是一道坎，在读题时一定要注意特殊情况，不要忘记考虑一些特殊情况，

1. 首先时**' . '**,它可以匹配任意一个单字符，就相当是一个万能符号
2. *其次就是**' * '**，这里注意他的原话是：” **匹配零个或多个前面的那一个元素**“，这里不要将其误解为单个字符就可以起作用的字符，他要发挥作用，还要依赖前面的字符，两者不单是大小或者说种类(a,b,c......)有关联，而且两者的数量，多少也是紧密相连的，最好的理解就是，**将前面的字符和' * '看成一个整体，举个示例来说，'a*',并不代表最少有一个a，它可以一个a都没有***，比如s=”asd“，p=”*dasd“，也是相匹配的， *d为一个整体，个数为0；

在读懂题意之后，我们就可以进行算法的设计了，相信在观察和模拟示例实现的过程中，我们也能够看出来，这个过程中最核心关键的部分就是带有*的数据要如何处理，是安排一个还是两个，还是多个呢，因为在扫描数据数据时我们是逐一扫描的，在扫描时我们是两个串的位置一同变化的，在扫描之前，我们并不知道s串中与带 *数据相同元素的个数是多少，而且就算扫描过后统计出来，先不说效率问题，单说代码实现方面，我们估计还要统计带 *数据后紧接着的与带 * 数据相同的元素的个数是多少，更何况我们还要考虑 *‘ . ’* 可以任何任意数据的情况，这一点的代码实现是相当复杂的，所以这里我们巧妙的**运用的dfs算法，将用多少个的问题转化为用一个或者不用的问题，就是在遇到带有 * 的数据时，dfs分出多路，不用就j+2，用了就i+1，然后分出去的dfs又会遇到同样的情况直到，不用带 * 数据或者全部数据都走了一遍，都不行的时候停止**

同时为了防止同样的数据反复执行，我们在加上记忆化dp，用二位dp保存每一对i，j的状态（这题的数据范围并不大），初始化为-1表示为走过的路，用0表示走不通，用1表示可行,这里记忆化的含义倒是与2770的记忆化含义差不多，即dp[i] [j]的含义从i，j出发是否能到达终点，节省反复计算的时间

还有一点需要注意，那就是dfs终止条件的设置，不能直接设置为i=n&&j=m时值为1，i == n&&j！=m || i！=n&&j == m时值为0，这样会漏掉p后面全是带 * 数据的结果，因为带 * 数据是可以匹配0个字符的，即相当于没有，这种情况也是可以到达终点的，所以这里就要考虑p的下一个数是带 * 数据的情况，如果是那就继递归 j+2，按照匹配0个字符来通过观察后来的结果，确定此时的状态



## 复杂度

- 时间复杂度:O($$n*m$$)
- 空间复杂度:O($$n*m$$)

## Code

```
class Solution {
public:
    bool isMatch(string s, string p) {
        int n=s.size();
        int m=p.size();
        vector<vector<int>> dp(n+1,vector<int>(m+1,-1));
        auto dfs=[&](auto self,int i,int j)->bool{
            
            if(dp[i][j]!=-1)return dp[i][j];
            if(i==n) return dp[i][j]= (j==m||(p[j+1]=='*'&&self(self,i,j+2)));
            if(j+1<m&&p[j+1]=='*'){
                bool skip=self(self,i,j+2);
                bool unskip=0;
                if(s[i]==p[j]||p[j]=='.'){
                    unskip=self(self,i+1,j);
                }
                return dp[i][j]=skip||unskip;
            }else{
                if(s[i]==p[j]){
                    return dp[i][j]=self(self,i+1,j+1);
                }else if(p[j]=='.'){
                    return dp[i][j]=self(self,i+1,j+1);
                }else{
                    return 0;
                }
            }
        };
        
        return dfs(dfs,0,0);
    }
};
```



# 方法二：纯dp

才疏学浅

前面方法的dp的特性显示并不明显，倒是在记忆化的方面比较凸显作用，而这个方法这是完全根据动态规划而来的，因为动态规划的种类较多，且状态方程的选取随着题目的变化而差距甚大，且就算是同一道题也可能会有多种状态方程可以使用，所以动态规划是相当难攻克的知识点

## 解题思路和过程

这里采用bool型的二位数组***dp[i] [j]来表示s串的i前边的字符串与j前边的字符串是否匹配***，true表示可以匹配，在知道dp表示的状态后，接下来就是状态方程的选取了，通过前面的分析（上一个方法）对数据的模拟，我们可以知道确定dp[i] [j]状态的关键难点在遇到带 * 字符的时候，这里有一个关键的问题处理办法就是，**将带 * 数据使用多少个的问题转化为：是否使用带 * 数据的问题**

因为dp[i] [j]状态的确定在遇到带 * 数据时依赖的情况较多，所以将多种情况拆开来讨论方便理解代码的实现。

在开始前我们不得不再次解释一下dp[i] [j]的含义是指s串的 **i 前边的字符串**与 **j 前边的字符串**是否匹配，这里我们可以基本看出，**dp状态的确定是依赖 i 前面的字符以及 j 前面的字符的**，所以在探测是否是带 * 数据是我们用的是p[j-1]确定状态时也用它，当我们遇到带 * 数据时，即**p[j-1]==' * '**时我们面对有两种情况：

1. 不使用带 * 数据
   - 如果不使用这个数据，那么就是当***dp[i] [j]就直接继承dp[i] [j-2]的状态***，这里可以**理解为继承了没有带 * 数据时的状态，变相的相当于跳过了这个带 * 数据**
2. 使用带 * 数据
   - 如果使用带 * 数据，在**p[j-2] == s[i-1]||p[j-2] == ' * '的前提下让dp[i] [j]继承dp[i-1] [j]的状态，或许这样说并不恰当，两个条件是必须同时满足的**

再然后就是**遇到普通数据或者 ' . '时**的情况，这个**只需要保证dp[i-1] [j-1]和p[j-2] == s[i-1]||p[j-2] == ' * '同时成立**即可

## 复杂度

- 时间复杂度:O($$n*m$$)

- 空间复杂度:O($$n*m$$)

## Code

```
class Solution {
public:
    bool isMatch(string s, string p) {
        int n=s.size()+1;
        int m=p.size()+1;
        vector<vector<bool>> dp(n,vector<bool>(m,0));
        dp[0][0]=1;
        for(int i=2;i<m;i+=2){
            dp[0][i]=dp[0][i-2]&&(p[i-1]=='*');
        }
        for(int i=1;i<n;++i){
            for(int j=1;j<m;++j){
                if(p[j-1]=='*'){
                    if(dp[i][j-2])dp[i][j]=1;
                    else if(dp[i-1][j]&&s[i-1]==p[j-2])dp[i][j]=1;
                    else if(dp[i-1][j]&&p[j-2]=='.')dp[i][j]=1;
                }else{
                    if((s[i-1]==p[j-1]||p[j-1]=='.')&&dp[i-1][j-1]){
                        dp[i][j]=1;
                    } 
                }
            }
        }

        return dp[n-1][m-1];
    }
};
```
