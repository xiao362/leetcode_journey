# 方法一：滑动窗口

## 解题思路

题目要求我们统计一个字符串 `x` 重新排列后，`word2` 是重排字符串的 前缀的子字符串的个数

> 注意：这里只是提到了重新排序不一定就是按照字典序来排，也就是说排序是尽量按照`word2`的形式来拍，换句话说，只要子串中对应元素的个数够了，那他就是满足条件的，这里就引申出来，只要一个子串满足条件，在往这个子串上加任何元素都是满足条件的

因为我们研究的对象是字符串，且与顺序没有太大关系，并且条件是可以量化的，那也就意味着我们可以将这个子串看成是一个窗口，用滑动窗口的思路来解题

我们让**窗口处于恰好不满足条件的状态**，即`[left,right]`是不满足条件的，但`[left-1,right]`，`[left-1,right]`，`[left-3,right]`......`[0,right]`都是满足条件的，**个数也就是left个**

因为条件是多个并集的形式，在一切情况下我们的比较条件可能来到26次，再叠加上本身的循环，虽然时间复杂度没变，但也是比较浪费时间的，所以我们将比较条件也量化换成满足条件的个数，以及需要满足的个数来判断

## 实现细节

1. `vector<int> require(26,0)`,required对应需要满足的条件个数
2. `vector<int> check(26,0)`，checked对应已经满足条件的个数
3. 用left，搭配while一直让**窗口处于恰好不满足条件的状态**
4. `ans+=left`来不断统计个数

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    long long validSubstringCount(string word1, string word2) {
        long long ans=0;
        int left=0;
        int checked=0;
        int required=0;
        vector<int> require(26,0);
        vector<int> check(26,0);
        for(int i=0;i<word2.size();++i){
            require[word2[i]-'a']++;
            if(require[word2[i]-'a']==1)required++;
        }
        for(int i=0;i<word1.size();++i){
            char ch=word1[i]-'a';
            check[ch]++;
            if(check[ch]==require[ch])checked++;
            while(checked==required){
                char p=word1[left]-'a';
                check[p]--;
                if(check[p]<require[p])checked--;
                left++;
            }
            ans+=left;
        }
        return ans;

    }
};
```
