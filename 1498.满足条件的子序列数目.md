# 方法一：双指针

## 解题思路

我们找到一个子数组，使得该子数组的最大值与最小值的和小于等于`target`，如果我们暴力找所有子数组，每一个成员用一个指针的化，时间复杂度会高出天际，所以我们只能同通过找两个最值以及在两个最值中间数（个数），来确定子序列的可能了，这也就意味着我们必须对原数组排序，这样我们只要找两个值（用双指针），最大值和最小值，至于子数组，只要是在这个区间内的值（通过坐标得到在这区间的个数）都可以用来形成子数组，比如在示例一中`[3,5,6]`这个数组是满足最大以及最小值的和小于等于target的，`[3,6]`，`[3,5]`，`[3,3]`就是再加上他本身就是包含`3`的时候可以行成的子数组

我们可以用双指针来找最大值和最小值以及，在着这个区间的元素的个数（这是统计子数组个数的条件），我们向上面一样通过以都包含最小值的子数组来逐一统计，倘若不这样，这个区间的数组还可以有`[5,6]`的话，当最小值移动是我们就会重复统计（二者和小于target时，左指针右移），以都包含左指针统计时，每次增加的个数是$2^k$，`k`是区间元素个数-1

## 实现细节

1. 因为每次增加的元素个数过大，很可能会溢出，所以别忘了取余

## 复杂度

- 时间复杂度:$O(n \log n)$，主要消耗在排序上
- 空间复杂度:$O(1)$

## Code

```C++
const int MOD=1'000'000'007;
const int MX=100'000;
int pow2[MX];
auto init=[]{
    pow2[0]=1;
    for(int i=1;i<MX;++i){
        pow2[i]=pow2[i-1]* 2 %MOD;
    }
    return 0;
}();

class Solution {
public:
    

    int numSubseq(vector<int>& nums, int target) {
        int ans=0;
        sort(nums.begin(),nums.end());
        int n=nums.size();
        int left=0;
        int right=n-1;
        while(left<=right){
            if(nums[left]+nums[right]<=target){
                ans+=pow2[right-left];
                ans%=MOD;
                left++;
            }else{
                right--;
            }
        }
        return ans;
    }
};
```
