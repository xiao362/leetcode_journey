# 方法一：字典树trie

## 解题思路

题目要求我们出最长的公共后缀，即从`wordsContainer` 中找与 `wordsQuery`的最长公共后缀，并要求我们：`wordsContainer` 中有两个或者更多字符串有最长公共后缀，那么答案为长度 **最短** 的。如果有超过两个字符串有 **相同** 最短长度，那么答案为它们在 `wordsContainer` 中出现 **更早** 的一个。

倘若使用暴力算法，时间复杂度最少是$O(n^2)$，且还没算上字符串长度，故时间复杂度会比这高很多，毫无疑问这会超时，为此我们要对算法优化，或者使用更好的算法，首先我们观察到，在暴力的模拟过程中，有一些前缀是相同的我们没必要重复查询，为此我们采用空间换时间大法，在后缀相同时把相同字符放在一起，在遇到不同字母时再分开，即采用树形结构来存数据，同时为了满足上面的要求，我们所开的结构体里，可以有一个数idx用来存储当前位置的最优`wordsContainer`元素的坐标

> 因为我们在建树时是可以从`wordsContainer`的前前面往后建的，所以出现 **更早** 的这个条件可以通过长度相同时不更新来跳过，我们只需要保证遇到的字符串与原先所存储的字符串（坐标）长度比较时，前者长度短时更新坐标即可

## 实现细节

1. 树里面每个字母存储时，可以固定长度为26，尽量节省空间`tree* ch[26]{}`
2. `void clear(tree* node)`用来释放空间
3. `int minidx=0`用来记录长度最小的被查询元素，用来再查询不到时返回
4. 通过`cur->ch[p]==nullptr`来判断是否存在某个字母，看是否需要new
5. `len<wordsContainer[cur->idx].size()`用来更新最优查询坐标，使每个位置都是最优的
6. `ans[i]=cur->idx`用来更新查询过程中每一步的最优坐标

> clear(root),千万不要忘记，不然就会爆空间

> 1. **不是评测系统“误判”**：评测系统忠实地捕捉到了极高的内存峰值。
> 2. **手动 `delete` 是根本解决之法**：在内存限制如此严格的情况下，如果不手动释放，内存占用会从 600 MB 继续飙升，导致不可接受的 MLE（内存超限）。你手动 `delete` 后，将内存回收到 609.5 MB 这个数值（虽然依然很高，但至少是可控且能在这个极限上运行通过的），这一步非常关键且必要。

## 复杂度

- 时间复杂度:$O(L1|\Sigma|+L2)$,L1是被查询数组字符串长度之和，反之则是查询数组字符串长度之和
- 空间复杂度:$O(L1|\Sigma|)$

## Code

```C++
class Solution {
public:
    struct tree{
        tree* ch[26]{};
        int idx;
    };

    void clear(tree* node){
        for(int i=0;i<26;++i){
            if(node->ch[i]){
                clear(node->ch[i]);
            }
        }
        delete node;
    }

    vector<int> stringIndices(vector<string>& wordsContainer, vector<string>& wordsQuery) {
        tree* root=new tree();
        tree* cur=root;
        int minidx=0;
        int n=wordsContainer.size();
        for(int i=0;i<n;++i){
            cur=root;
            int len=wordsContainer[i].size();
            if(len<wordsContainer[minidx].size()){
                minidx=i;
            }
            for(int j=len-1;j>=0;--j){
                int p=(wordsContainer[i][j]&31)-1;
                if(cur->ch[p]==nullptr){
                    cur->ch[p]=new tree();
                    cur=cur->ch[p];
                    cur->idx=i;
                }else{
                    cur=cur->ch[p];
                    if(len<wordsContainer[cur->idx].size()){
                        cur->idx=i;
                    }
                }
            }
        }
        int m=wordsQuery.size();
        root->idx=minidx;
        vector<int> ans(m,minidx);
        for(int i=0;i<m;++i){
            int len=wordsQuery[i].size();
            cur=root;
            for(int j=len-1;j>=0;--j){
                int p=(wordsQuery[i][j]&31)-1;
                if(cur->ch[p]==nullptr){
                    ans[i]=cur->idx;
                    break;
                }else{
                    cur=cur->ch[p];
                    ans[i]=cur->idx;
                }
            }
            
        }
        clear(root);
        return ans;
    }
};
```
