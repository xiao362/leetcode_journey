# 方法一：记忆统计法

**因为不用两个指针维护数组长度，所以时间上要比滑动窗口小一点，但二者的时间复杂度相同，这种差距也就微乎其微了**

## 解题思路和过程

通过题意我们可知题目要求我们求出**最长的只包含1的最长子数组的长度，并且在统计这个数组的时候必须删掉一个元素**

> *注意：**是必须删掉一个元素，没0就只能删1；***

这时我们观察数据可以想到使用滑动窗口来统计数组长度，也即是上免得题意**可以转化为里面最多存一个0，且这个0不算在数组长度里，**同样的思路我们先说另一种写法，

先**假设我们一定会遇到0**，设两个数a，b，其中**a统计当前遍历时遇到的全为1的数组长度**，**b记为上一个遇到的全为0的数组长度**，也就是**a遇到0后**，a从0继续开始计数，**把刚才的数给b**，这样也能统计时最多跳过去一个0，也能得到正确长度；

这时我们还要考虑倘若我们遇不到0呢，那我们就只能把原来的ans-1了，另外正如我说的我们只要统计是否能遇到0，遇不到就要-1，那我们是否可以接上面b的定义：b记为上一个遇到的全为0的数组长度，也就是b被赋值就遇到了0，不然就是没遇到，那**我们直接给他赋值为-1即可**（**也可以在单独开一个bool记录**），反正**计算最长长度是用的是a+b**

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int longestSubarray(vector<int>& nums) {
        int n=nums.size();
        int ans=0;
        int res=-1,cnt=0;
        for(int i=0;i<n;++i){
            if(nums[i]==1){
                cnt++;
            }else if(nums[i]==0){
                res=cnt;
                cnt=0;
            }
            ans=max(ans,cnt+res);
        }
        return ans;
    }
};
```

# 方法二：滑动窗口

## 解题思路和过程

接着上面滑动窗口的思路，我们可以**开一个窗口，不断维护区间里0的个数**，**只要区间里的0的个数大于1我们就要移动窗口的左指针知道出去一个0**，这一点是恰好符合题意的，简直是为他而设计的，因为这要我们就能直接用right-left得到数组长度了，因为不论数组中有有没有零我们都要删掉一个数，或者说我把他的逻辑给弄反了

我们换种说法也是换种思路，我们要求的是***最长子数组的长度（减一），满足子数组至多有一个 0***，我们要求是最长子数组的长度减一的最大值，并且这个数组中可以放一个0，这样就是一个好理解的滑动窗口了

接下来就是代码实现，用cnt记录窗口里0的个数，大于1就移动left直到cnt==1，原本距离是right-left+1，因为要-1，就变成了**right-left**，期间**别忘了用ans不断变换最大值即可**

## 复杂度

- 时间复杂度:$O(n)$,**最坏的情况每个元素走两遍**
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    int longestSubarray(vector<int>& nums) {
        int n=nums.size();;
        int ans=0;
        int cnt=0;
        int left=0;
        for(int i=0;i<n;++i){
            cnt+=1-nums[i];
            while(cnt>1){
                cnt-=1-nums[left++];
            }
            ans=max(ans,i-left);
        }
        return ans;
    }
};
```
