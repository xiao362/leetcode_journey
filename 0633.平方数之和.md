# 方法一：双指针

## 解题思路

如果我们事先不知道任何坐标位置，只是设两个指针在数组的两端，情况是否与上面类似呢，如果二者和偏大，这时我们就要让右指针左移，直到我们发现二者的和小于target了，这时我们发现，right右边的值是肯定不会成为我们需要的值，因为他们加最小值都大于target，然后接着如果我们想要等于target，我们就要使二者的和变大，也就是左指针右移（右指针右边的已经淘汰），直到二者和再次大于targte，此时类似的left左边的也已经被淘汰，因为他们加上数组可能值的最大值后还是小于target，以此类推，我们一定会遇到让二者和等于target的情况

> 注意：
>
> 1. 因为设计到平方所以可能会溢出，可以使用long long，或者不用让二者相加来而使用c-a*a的方法来判断二者相等，这样就能避免溢出
> 2. a==b时也可以取到

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    bool judgeSquareSum(int c) {
        long long right=sqrt(c);
        int left=0;
        while(left<=right){
            long long sum=left*left+(long long)pow(right,2);
            if(sum==c)return true;
            else if(sum>c) right--;
            else left++;
        }
        return false;
    }
};
```

# 方法二：枚举

## 解题思路

利用int遇到小数会向下取整的性质，我们枚举a，计算`sqrt(c - a * a)`是否会向下取整(在平方会来值会变小就说明向下取整了)

## 实现细节

a最大枚举到哪？

如果枚举到2a²>c时，仍然没有找到符合等式的a和b，则停止枚举，返回false。

为什么？此时a²>c-a²=b²，假如继续枚举能找到符合等式的a和b，比如a=5,b=3，那么之前在枚举到a=3时，也能发现a=3,b=5符合等式，矛盾。所以当枚举到2a²>c时，后面不可能找到符合等式的a和b。

对于C++等部分语言，如果直接计算2a²，可能会发生溢出。可以把循环条件2a²≤c改成等价的a²≤⌊c/2⌋，从而避免溢出。

## 复杂度

- 时间复杂度:$O(n)$
- 空间复杂度:$O(1)$

## Code

```C++
class Solution {
public:
    bool judgeSquareSum(int c) {
        for (int a = 0; a * a <= c / 2; ++a) {
            int b = sqrt(c - a * a);
            if (a * a + b * b == c)
                return true;
        }
        return false;
    }
};
```
