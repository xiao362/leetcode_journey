# 方法一：结构体排序+贪心



## 解题思路和过程

为了计算出完成任务后的最小初始能量，通过观察样例数据，我们可以对这整个的流程进行分析：

**为了得到最小的初始能量，我们可以将ans设为0，尽量将每一次添加的能量数维持到最小，那么最终我们得到的ans就是最小的初始能量值，这里是贪心的思路**，而为了实现这个算法，我们需要对这整个流程进行分析，即为了使每次添加的能量最小，我们需要做的事情，其问题就变相等价于我们剩下的能量最大，因为**剩下地能量多了我们需要添加的就少了。**

故我们可以对原来的数据actuali, minimumi的差值进行降序排序，而为了实现排序，我们不得不将数据的存储类型给改为结构体，不然会将同一组的数据给打乱，用结构体就能解决这个问题，再然后就是排序，**因为是对结构体排序且是降序，故我们不得不重写cmp排序规则（如果不用sort排序，可以自己写排序算法，也可以用lambda直接写）**

排完序后我们就可以，对数据进行处理，设置res不断更新完成任务后的剩余能量，（注意不是每次都要补能量，res有时是>minimum的）

## 复杂度

- 时间复杂度:($$nlogn$$),（快排的时间复杂度）
- 空间复杂度:($$n$$)

## Code

```

class Solution {
public:
    static struct task{
        int actual;
        int minimum;
    };
    static bool cmp(task a,task b){
        return a.minimum-a.actual>b.minimum-b.actual;
        
    }

    int minimumEffort(vector<vector<int>>& tasks) {
        int n=tasks.size();
        vector<task> t(n);
        for(int i=0;i<n;++i){
            t[i].actual=tasks[i][0];
            t[i].minimum=tasks[i][1];
        }
        sort(t.begin(),t.end(),cmp);
        int ans=0,res=0;
        for(int i=0;i<n;++i){
            if(res<t[i].minimum){
                ans+=t[i].minimum-res;
                res=t[i].minimum;
            }
            res-=t[i].actual;
        }
        return ans;
    }
};
```

# 方法二：lambda+贪心差值升序

不得不说原来终归还是我见识浅薄了，原来可以不用结构体直接对数据排序，即把二维数组的一行当作一个元素直接排序

## 解题思路和过程

同样把问题聚焦在每次添加的能量最小的方面上，在这之前我不得不提出**一个基本的事实:完成所有任务的最少初始能量绝对是大于等于actual消耗能量的总和的**，换句话说，我们可以***将完成所有任务的最少初始能量分为两部分***：

1. **actual消耗的能量和**
2. ***最终剩余能量(我想叫他门槛能量)这一部分能量并不参与完成任务的耗费，只是提供以个提高门槛的作用使任务能够开始完成，***那么这部分能量是由什么决定的呢，只是由minimum与actual的差值决定的吗，如果是那是否就代表我们直接找到最大差值将门槛拉到差值的最高（任务能全部完成的最低）就一劳永逸了呢，并不是，门槛能量取决于minimum与actual

我们要求的问题就可以转化为求最小门槛能量

这里我们可以将minimun与actual的差值递增排序，即完成任务后剩余的能量逐渐递增，我们设置ans为完成任务的最少能量，然后**不断迭代ans的值，（有点像动态规划的思想）**，在遍历时，ans即代表上一次完成任务的最少能量，我们每次都将actual能量加入到ans中然后与minimum比较，

1. **如果ans<minimum补充的能量就是门槛能量**
2. **如果ans>=minimum,那就说明当前的能量完全足够前面所有任务的完成**

这里在解释为什么要升序排列，因为我们要**确保每次剩余的能量能充分的利用**，尽可能降低门槛能量的补充

## 复杂度

- 时间复杂度:O($$nlogn$$)
- 空间复杂度:O($$n$$)

## Code

```
class Solution {
public:
    int minimumEffort(vector<vector<int>>& tasks) {
        sort(tasks.begin(),tasks.end(),[&](vector<int> &a,vector<int> &b){
            return a[1]-a[0]<b[1]-b[0];
        });
        int ans=0;
        for(auto task:tasks){
            ans=max(ans+task[0],task[1]);
        }
        return ans;
    }
};
```
